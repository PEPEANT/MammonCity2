# 밤 자동 취침 계획

현재 결론: 밤 배경 PNG를 따로 늘리지 않고 `22:00 도달 -> 검은 화면 자동 취침 -> 다음 턴 08:00 방 시작`으로 처리한다. `06:00`은 이번 계획에서 실제 플레이 시간대로 만들지 않고, 문구와 연출로만 표현한다.

## 목적

- 밤 10시 이후에도 그대로 플레이가 이어지는 흐름을 끊고, 야간을 강제 취침 구간으로 정리한다.
- 위치별 밤 PNG를 추가하지 않고도 밤이 되었다는 감각을 주는 최소 연출을 만든다.
- 플레이어가 어디에 있든 밤이 되면 집으로 돌아가 잠들고, 다음 턴 시작은 항상 집 안 방에서 고정한다.
- 자동 취침이 수익, 대출, 주식, 코인, 예약 출근, 월드 이동 상태와 충돌하지 않도록 선행 계획을 만든다.

## 현재 관련 동작

- 시간 시작/종료:
  - `js/data/constants.js`
  - 현재 하루는 `08:00` 시작, `24:00` 종료다.
- 턴 전환:
  - `js/logic.js`의 `advanceDayOrFinish()`가 다음 턴으로 넘기고, `prepareDayState()`가 새 턴 시작 상태를 만든다.
- 새 턴 시작 위치:
  - `prepareDayState()`는 이미 `scene = "room"`으로 만들고, `world.currentLocation`도 집 쪽 스폰 위치로 되돌린다.
- 월드 시간대:
  - `js/systems/world/world-runtime-service.js`의 `getWorldTimeBand()`는 이미 `21:00` 이후를 `night`로 본다.
  - 문제는 시간대 이름만 바뀌고 실제 게임 흐름은 `24:00`까지 계속 열려 있다는 점이다.
- 투자 보유분 계산:
  - 주식/코인 보유 수익률은 현재 `state.day` 기준 스냅샷으로 계산된다.
  - `prepareDayState()`가 시작될 때 `stocksUsedToday`, `coinUsedToday`가 리셋된다.
  - 즉 현재 구조는 `보유분 평가 = 턴당 1회`에 가깝다.

## 이번 계획에서 안 건드리는 것

- `06:00`부터 실제 플레이를 시작하는 시간표 개편
- 맵별 밤 PNG 제작
- 야간 전용 NPC/상점/배경 상태 확장
- 주식/코인에 별도의 `야간 추가 변동 1회`를 넣는 구조

위 네 가지는 이번 자동 취침 목표보다 범위가 훨씬 크다.

## 구현 계획 1~10

1. 밤 컷오프 기준을 `22:00`로 고정한다.
   - 현재 하루 시작이 `08:00`이므로, 이번 작업의 실질 규칙은 `22:00 이상이면 즉시 취침`이다.
   - `06:00`은 실제 시뮬레이션 시간이 아니라 다음 턴 아침을 설명하는 문구로만 남긴다.

2. 시간 소모 뒤 공통으로 호출되는 `강제 취침 판정` 헬퍼를 만든다.
   - 후보 위치:
     - `spendTimeSlots(...)`
     - `spendMinorTime(...)`
     - `advanceTimeToSlot(...)`
     - 필요 시 출근/미니게임 종료 후 시간 점프 지점
   - 핵심은 "어떤 행동이든 22:00를 넘기면 같은 방식으로 취침"하게 만드는 것이다.

3. 밤 연출은 새 배경 대신 `검은 화면 전용 씬`으로 만든다.
   - 새 씬 이름 예시: `night-auto-sleep`
   - 표시 문구 예시:
     - `밤이 되어 집으로 돌아가 잠을 잤다.`
     - `다음 날 아침, 방에서 눈을 뜬다.`
   - 이 씬은 월드 PNG를 참조하지 않고, 단색 또는 아주 어두운 오버레이만 사용한다.

4. 자동 취침은 `즉시 다음 턴 점프`가 아니라 `짧은 중간 연출 -> 다음 턴 전환` 순서로 처리한다.
   - 이유:
     - 사용자가 왜 턴이 넘어갔는지 이해하기 쉬움
     - 저장/불러오기 시 상태 원인을 추적하기 쉬움
     - 다음 턴 브리핑과 충돌이 적음
   - 권장 흐름:
     - 시간 소모
     - 22:00 이상 감지
     - `night-auto-sleep` 진입
     - 확인 또는 자동 1회 진행
     - `advanceDayOrFinish()`

5. 위치 복귀는 새 로직을 많이 만들지 말고 기존 `prepareDayState()` 복귀 로직을 그대로 탄다.
   - 밤이 되기 직전 위치가 어디든 직접 집 좌표를 하나하나 세팅하지 않는다.
   - 새 턴 시작 시 이미:
     - `scene = "room"`
     - `world.currentLocation = 집 스폰`
     - `world.currentDistrict = 집 구역`
   - 따라서 밤 취침 씬은 `다음 턴으로 넘기는 트리거`만 책임지는 편이 안전하다.

6. 강제 취침 직전 정리해야 하는 상태를 체크리스트로 묶는다.
   - `pendingTravel*`
   - `homeTransition`
   - `jobMiniGame`
   - `currentIncident`
   - `phoneStageExpanded`
   - `casinoVenueScreen`
   - 목표는 "밤에 이동 중/앱 확장 중/미니게임 중이던 상태가 다음 날 끌려오지 않게" 하는 것이다.

7. 수익/턴 시작 처리 순서를 문서 기준으로 고정한다.
   - 권장 순서:
     1. 마지막 행동 시간 반영
     2. 밤 자동 취침 연출
     3. `advanceDayOrFinish()`
     4. 체력/에너지 회복
     5. `prepareDayState()`
     6. 대출 자동 상환
     7. 예약 출근/직장 결과/복권/고정 수익 브리핑
     8. 새 턴 시장 스냅샷 반영
   - 이렇게 해야 "밤 연출"과 "다음 턴 정산"이 섞이지 않는다.

8. 주식/코인 보유분은 `야간 추가 변동`을 넣지 않고, 계속 `턴당 1회 평가` 원칙을 유지한다.
   - 현재 구조에서 보유 수익률은 이미 `state.day` 기반이다.
   - 여기서 밤 자동 취침 때 별도 야간 수익률을 한 번 더 굴리면:
     - 다음 턴 시장 변화와 중복 계산
     - 손익이 실제보다 두 번 반영
     - 사용자가 손익 근거를 이해하기 어려움
   - 결론:
     - 자동 취침은 `시장 평가 시점`을 추가하지 않는다.
     - 시장 평가는 계속 "다음 턴 시작 시점" 한 번만 바뀐다.

9. 보유 중인 주식/코인의 "밤새 흔들린 느낌"은 숫자 추가 계산이 아니라 UI 설명으로 준다.
   - 예시:
     - 다음 턴 브리핑에서 `밤사이 시장 분위기가 뒤집혔다`
     - 주식 앱 홈에서 `전 턴 대비 시가 갭`
     - 코인 앱에서 `야간 과열/야간 투매` 헤드라인
   - 즉 감각은 강화하되, 실제 손익 공식은 하나만 유지한다.

10. QA는 시간/투자/이동 꼬임 위주로 묶어서 본다.
   - 최소 확인 항목:
     - `21:30 -> 22:00` 진입 시 자동 취침 발생
     - 바깥/이동 중/집 복도/폰 펼치기 상태에서 모두 정상 취침
     - 다음 턴 시작이 항상 `room`
     - `world.currentLocation`이 집 스폰으로 복귀
     - 주식 보유 중 자동 취침 후 손익이 1회만 바뀜
     - 코인 보유 중 자동 취침 후 손익이 1회만 바뀜
     - 대출 자동 상환/복권/직장 브리핑이 한 번씩만 뜸

## 금융 영향 정리

### 현재 권장안

- 밤 자동 취침은 `행동 가능 시간 단축`만 만든다.
- 주식/코인 가격 공식은 그대로 둔다.
- 보유 자산 손익은 다음 턴 시작 시점의 시장 스냅샷만 본다.

### 이렇게 하는 이유

- 지금 게임은 `1턴 = 1개월` 성격이 강하다.
- 밤 22:00 연출은 "월 안의 남은 시간 정리"에 가깝지, "월을 하나 더 세분화한 야간 세션"이 아니다.
- 야간 세션을 따로 넣으면:
  - 주식/코인 변동이 실제 턴보다 더 자주 발생
  - 강제 취침이 사실상 투자 가속 버튼이 됨
  - 월급, 대출, 자산, 뉴스 캘린더 해석이 꼬인다

### 나중에 확장하고 싶을 때

- `야간 추가 수익률`이 아니라 `갭 오프닝 연출`을 먼저 넣는다.
- 필요하다면 훗날 아래처럼 분리한다.
  - 낮 세션: 행동/매수/매도 가능
  - 야간 세션: 행동 불가, 가격 연출만 표시
  - 다음 턴 시가: 야간 결과를 반영
- 하지만 이건 이번 작업 범위를 넘는 별도 시장 시스템 개편이다.

## 완료 조건

- 22:00를 넘기면 장소와 무관하게 밤 자동 취침 연출이 뜬다.
- 새 턴은 항상 집 안 방에서 시작한다.
- 밤용 위치 PNG 추가 없이도 플레이 흐름이 자연스럽다.
- 주식/코인 보유 손익이 야간에 중복 계산되지 않는다.
- 다음 세션에서 구현자가 바로 착수할 수 있을 정도로 순서와 리스크가 문서에 정리돼 있다.
