# DEV 편집 도구
현재 결론: `game.html?dev=1`에서는 DEV PANEL이 작은 도크만 남고, 각 기능이 독립 플로팅 창으로 뜬다. `일차 이동`, `행동 / 스탯`, `이벤트 재생`, `위치 편집`, `나레이션 편집`, `현재 상태`를 각각 따로 열고 닫을 수 있다.

## 목적

- 게임 화면을 보면서 바로 장면, 좌표, 텍스트를 테스트한다.
- 실제 코드 파일은 자동 수정하지 않고, 브라우저 로컬 저장만 사용한다.
- 본 저장과 분리된 DEV 미리보기 상태로 이벤트를 점검한다.

## 현재 동작

1. `game.html`은 `index.html?dev=1`을 열고 DEV 도구를 같이 띄운다.
2. 각 DEV 기능은 `body` 기준의 드래그 가능한 독립 창으로 분리된다.
3. 창 위치와 접힘 상태는 `localStorage`에 저장된다.
4. `위치 편집`은 현재 화면에 보이는 배우나 청소 아이템만 드래그할 수 있다.
5. 배우와 아이템은 화면 경계 밖이나 나레이션 아래로 조금 빼서 둘 수 있다.
6. 위치 편집은 현재 드래그 중심으로 사용하며, 세부 수치 편집 UI는 숨김 상태로 둔다.
7. `나레이션 편집`은 현재 장면의 화자, 제목, 본문을 로컬 오버라이드로 덮어쓴다.
8. `이벤트 재생`은 날짜별 `dev preset`을 읽어서 깨끗한 테스트 상태로 장면을 연다.
9. `이벤트 재생` 중에는 `devPreviewMode`가 켜지고 본 저장은 갱신되지 않는다.
10. `원래 상태`를 누르면 재생 전 상태로 복구된다.

## 로컬 저장 키

- 창 위치와 접힘 상태: `mammon-city-dev-sections-v1`
- 위치 편집 배치 저장: `mammon-city-dev-layout-v1`
- 나레이션 편집 오버라이드: `mammon-city-dev-text-v1`

## 안전 규칙

- DEV 도구는 코드 파일을 자동으로 수정하지 않는다.
- 위치와 나레이션 편집은 현재 브라우저의 `localStorage`에만 저장된다.
- 이벤트 재생도 본 저장 슬롯을 덮어쓰지 않는다.
- 실제 코드 반영은 `복사`로 나온 값이나 day 데이터 preset을 직접 파일에 붙여넣을 때만 일어난다.
