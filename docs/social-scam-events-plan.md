# 사이비 / 접근형 사기 이벤트 계획

현재 결론: `사이비 권유`와 `번호 접근 사기`를 바깥 월드의 `선제 접근 이벤트`로 묶고, v1은 미니게임 없이 `접근 -> 선택 -> 즉시 결과` 흐름으로 구현한다.

## 목적

- 바깥 이동에 `돈 벌 기회`만 아니라 `사람 때문에 생기는 손해`도 넣는다.
- 흙수저 + 외모 1 초반 루트에 체감되는 사회적 리스크를 추가한다.
- 기존 NPC 대화 시스템을 크게 늘리지 않고도 짧고 강한 사건을 만든다.
- 향후 `사기`, `유혹`, `권유`, `허위 계약`류 이벤트를 같은 방식으로 확장할 수 있게 한다.

## 현재 규칙

- 이벤트는 둘 다 `플레이어가 먼저 거는 대화`가 아니라 `NPC가 먼저 다가오는` 구조다.
- 둘 다 바깥 월드에서만 뜬다.
- 둘 다 `선택지 1번 -> 바로 손해`로 끝나는 짧은 사건으로 시작한다.
- 문서/코드 용어는 비하 표현 대신 아래 이름으로 통일한다.
  - `cult-recruiter`: 사이비 권유자
  - `romance-scammer`: 번호 접근 사기꾼
- `npc_4.png`, `npc_2.png`는 현재 프로토타입 소스 아트로만 본다.
- 구현할 때는 이미 쓰는 일반 NPC와 같은 ID를 재사용하지 않는다.
  - `npc_4.png`는 이미 일반 캠퍼스 NPC 계열로 쓰이므로 `cult-recruiter` 전용 런타임 복사본을 따로 둔다.
  - `npc_2.png`도 일반 학생 계열과 분리해 `romance-scammer` 전용 런타임 복사본을 둔다.

## 이벤트 1: 사이비 권유

### 핵심 루프

1. 플레이어가 집 앞, 골목, 사거리, 역 앞, 중심가 같은 바깥 위치에 들어간다.
2. `tryStartAmbientScamEvent()`가 조건을 확인한다.
3. `cult-recruiter`가 먼저 말을 건다.
4. 플레이어가 `같이 간다 / 사양한다 / 무시한다` 중 하나를 고른다.
5. `같이 간다`를 고르면 짧은 컷씬 뒤 `헌금을 뜯겼다` 결과가 뜬다.
6. `10만원`이 현금 우선, 부족하면 계좌에서 차감된다.

### 우선 조건

- 흙수저일수록 잘 뜬다.
- 외모 레벨 1일수록 잘 뜬다.
- 밤 자동취침 직전, 미니게임 중, 직장 중, 전화 앱 내부에서는 뜨지 않는다.

### 권장 발생 규칙

- 대상 위치:
  - `apt-alley`
  - `silver-home-front`
  - `golden-home-front`
  - `city-crossroads`
  - `station-front`
  - `downtown`
- 시간대:
  - `day`
  - `evening`
- 발생 확률:
  - 흙수저 + 외모 1: `8%`
  - 흙수저 + 외모 2 이상: `4%`
  - 기타 시작 + 외모 1: `3%`
  - 그 외: `1%`
- 쿨다운:
  - 최소 `5일`
- 일일 제한:
  - 하루 최대 `1회`

### 결과

- 수락:
  - `헌금을 뜯겼다`
  - `100000원` 차감
  - 메모리 추가
  - 헤드라인 추가
- 거절:
  - 금전 손실 없음
  - 짧은 불쾌/압박 문구만 남김
- 무시:
  - 금전 손실 없음
  - 추후 같은 지역 확률을 아주 조금만 유지

### 프로토타입 대사

- 접근:
  - `저기요, 얼굴이 너무 지쳐 보이시는데 잠깐 같이 가보실래요?`
  - `가까운 곳에 좋은 모임이 있는데 오늘만 잠깐 오시면 돼요.`
- 결과:
  - `정신을 차리고 보니 낯선 예배당 의자에 앉아 있었다.`
  - `거절하기 어려운 분위기에 휩쓸려 헌금 10만원을 냈다.`

## 이벤트 2: 번호 접근 사기

### 핵심 루프

1. 플레이어가 사거리, 역 앞, 중심가, 디지털단지 입구 같은 유동 지역에 들어간다.
2. 확률이 맞으면 `romance-scammer`가 먼저 접근한다.
3. `저기 혹시 너무 잘생기셔서 그런데 번호 좀 주실 수 있을까요?` 식으로 먼저 말을 건다.
4. 플레이어가 `번호를 준다 / 사양한다` 중 하나를 고른다.
5. `번호를 준다`를 고르면 짧은 몽타주로 `몇 번 만난 뒤 함정 합의금을 뜯겼다` 결과로 넘어간다.
6. `50만원`이 현금 우선, 부족하면 계좌에서 차감된다.

### 현재 결정

- 외모와 무관하게 뜬다.
- 다만 매우 가끔만 뜬다.
- v1은 실제 연애 루트를 열지 않는다.
- v1은 `가벼운 호감 이벤트`가 아니라 `접근형 사기 이벤트`로 고정한다.

### 권장 발생 규칙

- 대상 위치:
  - `city-crossroads`
  - `station-front`
  - `downtown`
  - `office-plaza`
- 시간대:
  - `day`
  - `evening`
- 발생 확률:
  - 기본 `0.8%`
- 쿨다운:
  - 최소 `10일`
- 일일 제한:
  - 하루 최대 `1회`
- 전체 제한:
  - 사이비 권유와 합쳐 하루 `선제 접근 사기 이벤트`는 `1회`만 허용

### 결과

- 수락:
  - `연락을 주고받다가 함정에 걸렸다`
  - `합의금 명목으로 500000원` 차감
  - 메모리 추가
  - 헤드라인 추가
- 거절:
  - 금전 손실 없음
  - 접근이 어색하게 끝난다

### 프로토타입 대사

- 접근:
  - `저기 혹시 너무 잘생기셔서 그런데 번호 좀 주실 수 있을까요?`
- 결과:
  - `연락은 빠르게 가까워졌고, 사건은 더 빠르게 꼬였다.`
  - `결국 합의금 명목으로 50만원을 뜯겼다.`

```js
state.socialScams = {

    outcome: "",
    amount: 0,
  },

```

## 공통 처리 규칙

- 돈 차감은 `현금 -> 계좌` 순서로 처리한다.
- 어떤 경우에도 잔액이 음수로 내려가면 안 된다.
- 플레이어가 보유한 유동자금이 너무 적어도 이벤트 자체는 뜰 수 있다.
- 단, 결과 연출은 `뜯길 수 있는 만큼`만 빼지 않고, 목표 금액을 우선 차감한다.
- 따라서 차감 헬퍼는 `cash + bank >= 목표 금액` 여부를 먼저 확인해야 한다.
- 목표 금액보다 적으면 다음 중 하나로 통일해야 한다.
  - 추천: 이벤트 자체를 막는다.
  - 비추천: 부족한 만큼만 뺀다.
- 이번 문서 기준 v1 권장안은 `차감 가능할 때만 발동`이다.

## 연출 규칙

- 접근 장면은 `dialogue` 또는 `incident` 기반으로 짧게 시작한다.
- 돈이 빠지는 순간은 별도 결과 카드로 강조한다.
- 결과 문구는 `불쾌한 사건`으로 읽히게 만들고, 유머톤으로 처리하지 않는다.
- 메모리 태그는 아래 둘로 시작한다.
  - `사이비`
  - `접근 사기`

## 구현 순서 1~10

1. 전용 기획 문서 기준으로 용어를 고정한다.
2. `npc_4.png`, `npc_2.png`를 전용 사기 NPC 런타임 자산으로 분리한다.
3. `cult-recruiter`, `romance-scammer` 전용 NPC ID를 추가한다.
4. `state.socialScams`를 저장/불러오기 대상에 넣는다.
5. 월드 이동 후 호출되는 `선제 접근 이벤트 resolver`를 추가한다.
6. 사이비 권유 이벤트를 먼저 구현한다.
7. 번호 접근 사기 이벤트를 같은 레지스트리 구조로 구현한다.
8. 차감 헬퍼와 헤드라인/메모리 추가를 공통 유틸로 묶는다.
9. 하루 제한, 이벤트별 쿨다운, 우선순위 예외를 붙인다.
10. 실제 플레이에서 확률과 위치를 조정한다.

## 리스크와 제약

- 같은 얼굴이 일반 학생 NPC와 사기 이벤트 NPC로 동시에 보이면 몰입이 깨진다.
- 초반 돈 손실 이벤트가 너무 자주 뜨면 게임이 억까처럼 느껴진다.
- `꽃뱀` 같은 표현을 식별자와 문서명에 쓰면 이후 유지보수 톤이 거칠어진다.
- 기존 로맨스/접근 이벤트와 우선순위가 겹치면 플레이어가 무엇이 보상이고 무엇이 함정인지 헷갈릴 수 있다.
- 돈 차감 이벤트는 로또, 카지노, 자동취침, 면접, 강연 결과 같은 우선 장면을 끊으면 안 된다.

## 완료 조건

- 바깥 월드에서 두 종류의 선제 접근 이벤트가 뜬다.
- 사이비 권유는 흙수저 + 외모 1에서 체감상 더 잘 뜬다.
- 번호 접근 사기는 외모와 무관하게 드물게 뜬다.
- 수락 시 각각 `10만원`, `50만원` 차감이 정상 작동한다.
- 하루 제한과 쿨다운 때문에 같은 날 연속으로 돈을 뜯기지 않는다.
- 일반 로맨스 이벤트와 사기 이벤트가 같은 얼굴/같은 ID를 공유하지 않는다.
