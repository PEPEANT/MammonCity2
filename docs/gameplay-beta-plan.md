# 베타 플레이성 재정비
현재 결론: 베타에서 먼저 살릴 건 `무엇을 해야 하는지 바로 보이는 목표`, `한 턴 안에서 돈과 계층 준비가 동시에 굴러가는 선택`, `턴 종료 때 성과가 확실히 느껴지는 보상`이다.

## 목적

지금 MammonCity는 시스템 수가 많아졌지만, 플레이어가 `지금 뭘 해야 하는지`, `왜 그걸 해야 하는지`, `이번 턴에 무엇이 달라졌는지`를 바로 읽기 어렵다.  
베타 단계에서는 기능을 더 넓히기보다, 이미 들어간 직장/알바/지도/시장/은행 루프를 게임처럼 읽히게 만드는 쪽이 우선이다.

## 베타 코어 루프

1. 게임이 현재 목표를 먼저 보여준다.
2. 플레이어는 이번 턴에 `생활비 확보`와 `직장 준비` 중 어디에 시간을 쓸지 고른다.
3. 장소 이동과 앱 행동이 시간을 먹으며, 행동 하나하나가 턴 압박으로 이어진다.
4. 턴 끝에는 돈, 준비도, 자격, 재직 상태, 자산 변화가 다음 선택을 바꾼다.
5. 12턴 종료 시 랭킹과 엔딩이 그 누적 선택을 요약한다.

## 우선순위

### 1. 목표 신호층

- 게임이 `지금 할 일`을 한 줄로 계속 보여줘야 한다.
- 예약 출근, 생활비 부족, 졸업 준비, 자격증, 직장 지원 같은 다음 행동을 HUD와 폰 홈에서 바로 읽게 만든다.
- 현재 반영:
  - 방/바깥/카지노 현장에 현재 목표 프롬프트 표시
  - 폰 홈 기본 미리보기가 현재 목표를 우선 표시

### 2. 한 턴 안의 선택 압축

- 이번 턴 행동은 많아 보이기보다 `3개 안쪽의 의미 있는 선택`으로 읽혀야 한다.
- 베타에서는 `알바`, `직장 준비`, `자산 관리` 3축만 선명하게 보이게 유지한다.
- 앱과 버튼 문구는 최대한 짧게 두고, 다음 행동을 헷갈리게 하는 장식 정보는 줄인다.

### 3. 장소별 개성 강화

- 같은 돈벌이여도 `어디에서 버는지`가 달라야 기억에 남는다.
- 배금역, 대학가, 배금디지털단지, 맥도날드, 물류센터, 카지노처럼 이미 만든 장소를 `목표 -> 이동 -> 현장 행동 -> 퇴근/결과` 흐름으로 더 선명하게 만든다.
- 특히 맥도날드는 `외부 -> 카운터 -> 주방` 구조와 `카운터 / 조리` 2포지션 차이를 베타 개성 포인트로 삼는다.
- 다음 구현 후보:
  - 배금역에서 열리는 중장거리 루트
  - 맥도날드 카운터와 주방의 리듬 차이를 더 강하게 보여주는 연출
  - 대학가 준비 행동 결과를 더 강하게 보여주는 연출
  - 직장 내부 NPC/이벤트

### 4. 턴 보상 강화

- 턴이 끝날 때 `돈만 올랐다`보다 `이번 턴에 인생이 한 칸 올라갔다`는 느낌이 필요하다.
- 출근 결과, 자격증 획득, 졸업, 정장 구매, 직장 합격처럼 상태가 바뀐 순간을 더 강하게 보여준다.
- 엔딩과 랭킹도 최종 보유 자금뿐 아니라 직업/수저/핵심 선택이 같이 기억나게 다듬는다.

## 리스크

- 목표 안내가 너무 길어지면 또 읽지 않게 된다.
- 목표가 많아지면 오히려 플레이어를 다시 헷갈리게 만든다.
- 베타에서는 `이번 턴 바로 할 행동`만 안내하고, 장기 목표는 폰/캐릭터 패널에서만 보조하는 정도가 적당하다.
