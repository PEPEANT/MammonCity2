# 기억 시스템

결론: 스마트폰 아래에 `기억` 버튼을 따로 두고, 특정 이벤트 결과를 30일 동안 유지되는 `기억`으로 저장해서 면접 확률, 특수 이벤트 해금, 범죄 루트 진입 조건에 연결한다.

## 목적

- 플레이어가 겪은 사건이 그냥 1회성 연출로 끝나지 않게 만든다.
- 중요한 경험을 `기억`이라는 형태로 저장해서 이후 선택지와 확률에 영향을 주게 한다.
- 스마트폰 UI 아래에 별도 진입 버튼을 두어, 플레이어가 현재 쌓인 기억과 숨은 분기 조건을 직관적으로 확인하게 만든다.

## 현재 방향

### 핵심 규칙

1. `기억`은 이벤트 완료 후 즉시 획득된다.
2. 한 번 획득한 기억은 해당 플레이의 30일 동안 사라지지 않는다.
3. 기억은 다음 회차로 계승하지 않는다.
4. 기억은 단순 수집 요소가 아니라 `확률 보정`, `이벤트 해금`, `특수 선택지 개방`에 직접 연결된다.
5. 기억 UI는 스마트폰 앱 목록 아래에 붙는 별도 버튼으로 연다.
6. 기억 UI는 스마트폰 본체와는 구분되는 별도 패널로 연출한다.

### 기억의 역할

- 면접 성공 확률 증가
- 특정 직업 지원 시 숨은 가산점 부여
- 범죄/욕망/집착 계열 이벤트 해금
- 관계/트라우마/집념 계열 분기 활성화
- 특정 엔딩 조건 누적

## UI 계획

### 위치

- 스마트폰 패널 하단에 `기억` 버튼을 추가한다.
- 기존 앱 버튼과 섞지 않고, 하단 고정 버튼처럼 둔다.
- 버튼 아이콘은 `폴더`, `서랍`, `기억 조각` 중 하나로 잡는다.

### 동작

1. 플레이어가 스마트폰 하단의 `기억` 버튼을 누른다.
2. 별도의 `기억 패널`이 열린다.
3. 패널 안에서 획득한 기억 카드 목록을 본다.
4. 각 카드는 이름, 획득 일차, 출처 이벤트, 현재 효과를 보여준다.

### 화면 구성

- 상단: `기억` 제목
- 본문: 획득한 기억 카드 리스트
- 카드 요소:
  - 기억명
  - 짧은 설명
  - 획득 일차
  - 출처
  - 실제 효과

### 표시 원칙

- 아직 획득하지 않은 기억은 기본적으로 숨긴다.
- 숨겨진 시스템을 너무 노출하지 않기 위해, 효과 문구는 완전 수치보다 `청소부 면접에서 유리해짐` 같은 식으로 쓴다.
- 악성 기억도 동일한 형식으로 저장하되, 카드 톤만 다르게 준다.

## 시스템 흐름

```text
이벤트 발생
  -> 결과 선택
  -> 기억 획득 조건 충족 확인
  -> 기억 추가
  -> 기억 효과 적용용 플래그/태그 저장
  -> 이후 면접/이벤트/분기에서 기억 조건 검사
```

## 기억 데이터 구조 계획

```js
{
  id: "clean-room-memory",
  title: "방을 치운 손",
  category: "survival",
  sourceEvent: "day1-cleanup",
  obtainedDay: 1,
  description: "몸을 움직여 어질러진 방을 직접 정리했다.",
  effects: {
    interviewBonus: {
      janitor: 0.15
    }
  },
  tags: ["cleaning", "discipline"]
}
```

### 런타임 상태값 계획

- `state.memories`
  - 현재 회차에서 획득한 기억 목록
- `state.memoryLookup`
  - 빠른 조건 검사 용도
- `state.memoryTags`
  - 이벤트 해금용 태그 모음

## 기억 획득 예시

### 예시 1. 방청소

- 트리거
  - 1일차 방청소 미니게임 완료
- 획득 기억
  - `방을 치운 손`
- 효과
  - 청소부 지원 시 면접 통과 확률 증가
  - 성실함 계열 태그 획득

### 예시 2. 여자에게 차임

- 트리거
  - 관계 이벤트에서 거절/이별 결과 발생
- 획득 기억
  - `비어 있는 자리`
- 효과
  - 성공과 돈에 대한 집착 계열 태그 획득
  - 사기, 조작, 불법 영업 같은 범죄 이벤트 풀이 열릴 수 있음
  - 냉소적 선택지 일부 해금

### 예시 3. 면접 연속 탈락

- 트리거
  - 특정 횟수 이상 면접 낙방
- 획득 기억
  - `떨어지는 감각`
- 효과
  - 위축 계열 이벤트 해금
  - 반대로 절박함 기반 무리수 선택지도 열릴 수 있음

## 효과 적용 방식 계획

### 1. 확률 보정

- 특정 기억이 있으면 특정 면접에 가중치를 더한다.
- 예:
  - `방을 치운 손` 보유 시 `청소부` 면접 합격률 `+15%`

### 2. 태그 해금

- 기억이 직접 숫자를 올리는 대신 태그만 부여할 수도 있다.
- 예:
  - `비어 있는 자리` 획득 시 `greed`, `resentment` 태그 추가

### 3. 이벤트 조건

- 이벤트 데이터에서 `requiredMemoryIds` 또는 `requiredTags`를 검사한다.
- 예:

```js
{
  id: "fraud-opportunity",
  requiredTags: ["greed"],
  blockedTags: ["stable-life"]
}
```

## 분기 설계 원칙

1. 기억은 무조건 좋은 버프만 주지 않는다.
2. 기억은 `능력`, `상처`, `욕망`, `집착`을 모두 포함한다.
3. 좋은 기억과 나쁜 기억이 동시에 다른 루트를 열 수 있어야 한다.
4. 기억이 쌓일수록 플레이어 인생 방향이 더 선명해져야 한다.
5. 기억은 엔딩 판정에도 일부 반영할 수 있다.

### 파일 역할

- `memories-data.js`
  - 기억 정의 목록
- `memory-effects-data.js`
  - 직업/이벤트에 붙는 기억 조건 데이터
- `memory-ui.js`
  - 스마트폰 하단 버튼, 기억 패널 렌더링
- `memory-logic.js`
  - 기억 획득, 중복 방지, 조건 검사
- `memory.css`
  - 기억 카드와 패널 전용 스타일

## 구현 순서 제안

1. 스마트폰 하단에 `기억` 버튼 UI만 추가
2. `state.memories`와 기억 추가 함수 구현
3. 1일차 방청소 완료 시 첫 기억 저장
4. 기억 패널 카드 UI 구현
5. 청소부 면접 확률 보정 연결
6. 관계 실패 -> 범죄 이벤트 해금 기억 추가

## 제약

- 30일 동안 영구 저장이므로, 같은 기억 중복 획득은 막아야 한다.
- 기억이 너무 많아지면 패널이 난잡해지므로 카테고리나 정렬 기준이 필요하다.
- 수치가 너무 노골적이면 메타 공략이 쉬워지므로 효과 설명 문구는 절제한다.
- 범죄 이벤트 해금은 단일 기억 하나보다 `기억 + 태그 + 현재 자금/상태`를 함께 보는 쪽이 안전하다.
