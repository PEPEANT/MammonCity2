# 소비 시스템

현재 결론: 소비는 단순 돈 삭제 장치가 아니라 `스탯 상승 + 인벤토리 획득 + 소유 자산 변화 + 생활 유지비`를 동시에 담당하는 성장 시스템이어야 한다. 플레이어는 돈을 모으기만 하는 게 아니라, 어디에 소비하느냐에 따라 전혀 다른 사람이 되어야 한다.

## 목적

- 소비를 `식비, 월세` 같은 생존비만이 아니라 성장 루프로 확장한다.
- 부동산, 자동차, 옷, 휴대폰, 외모, 운동 소비가 각각 다른 보상을 주게 만든다.
- 소비 결과가 반드시 `스탯 / 인벤토리 / 소유 시스템` 중 하나 이상에 남게 만든다.
- 돈을 버는 이유가 단순 생존이 아니라 `계급 상승`, `외형 변화`, `직업 해금`, `이동 효율`로 이어지게 만든다.

## 핵심 원칙

1. 모든 소비는 최소 하나의 `지속 효과`를 남겨야 한다.
2. 소비는 `오늘만 좋은 선택`과 `장기적으로 좋은 선택`이 섞여 있어야 한다.
3. 비싼 소비일수록 `인벤토리`, `ownership`, `appearance` 같은 상태에 흔적이 남아야 한다.
4. 소비는 직업과 연결돼야 한다. 잘 입고, 잘 꾸미고, 잘 쉬고, 잘 이동할수록 더 좋은 일을 얻어야 한다.
5. 소비가 없으면 생존이 힘들고, 소비만 하면 현금흐름이 꼬이게 만들어 긴장감을 유지한다.

## 소비 4계층

```text
생존 유지비
  -> 안 쓰면 바로 페널티

능력 투자 소비
  -> 지능, 체력, 외모, 평판 같은 성장

생활 수준 소비
  -> 집, 차, 폰, 옷처럼 일상 편의와 사회적 체급 상승

과시 / 사치 소비
  -> 즉시 만족감과 평판은 크지만 유지비와 손실 리스크도 큼
```

## 현재 코드와 연결할 축

현재 상태 기준으로 소비는 아래 상태와 먼저 연결하는 것이 가장 안전하다.

- `state.inventory`
  구매한 휴대폰, 의류, 백화점 품목, 소모품, 액세서리 보관
- `state.ownership`
  집, 차량 같은 대형 자산 보유
- `state.appearance.attractiveness`
  외모꾸미기, 옷, 성형, 운동 결과 누적
- `state.happiness`
  소비 만족감, 생활 수준 변화 반영
- `state.stamina`, `state.energy`
  운동, 식비, 거주환경, 차량 보유 효과 반영
- `지능 / 평판 / 범죄도`
  직업 해금, 사회적 평가, 범죄 루트 조건 반영

## 소비 카테고리

상세 소비는 도시 안의 실제 시설과 연결돼야 한다. 시설 설계는 `docs/design/city-facilities.md`를 함께 기준으로 본다.

### 1. 부동산 소비

핵심 역할: `거처 업그레이드 + 회복 효율 + 생활 수준 상승`

#### 소비 예시

- 고시원 입주
- 원룸 월세 계약
- 오피스텔 보증금/관리비 지불
- 아파트 매매
- 인테리어 / 가전 업그레이드

#### 연결 효과

- `state.ownership.home`, `state.ownership.residence` 변경
- 인벤토리 `문서` 탭에 계약서, 등기, 관리비 고지서 추가
- 높은 거처일수록 수면 회복량 증가
- 높은 거처일수록 행복도, 평판 보너스
- 일부 고급 직군/NPC 루트 해금

#### 유지비

- 월세
- 관리비
- 수리비
- 보증금 손실 / 전세사기 리스크

#### 기대 효과 예시

| 소비 단계 | 효과 |
| --- | --- |
| 고시원 | 최소 독립, 행복도 소폭 상승 |
| 원룸 | 에너지 회복량 증가, 준비 행동 효율 상승 |
| 오피스텔 | 평판 보너스, 일부 사무직/연애 루트 보정 |
| 아파트 이상 | 평판 대폭 상승, 상위 직군/NPC 접근 보너스 |

### 2. 자동차 소비

핵심 역할: `이동 효율 + 특정 직업 보너스 + 사회적 체면`

#### 소비 예시

- 자전거 구매
- 중고차 구매
- 외제차 구매
- 차량 정비
- 기름값, 보험료, 주차비 지출

#### 연결 효과

- `state.ownership.vehicle` 변경
- 인벤토리 `소지품` 탭에 차량 키 자동 생성
- 이동 시간 단축
- 배달/운송/영업형 직업 보너스
- 차량 등급에 따라 평판 또는 외모 연출 보정

#### 유지비

- 주유비
- 보험료
- 수리비
- 주차비

#### 기대 효과 예시

| 차량 | 효과 |
| --- | --- |
| 자전거 | 배달 진입, 체력 소모 소폭 감소 |
| 중고차 | 장거리 이동 효율 상승, 일부 일자리 해금 |
| 외제차 | 평판/과시 효과 크지만 유지비 부담 큼 |

### 3. 백화점 소비

핵심 역할: `프리미엄 상점 허브`

백화점은 하나의 상품이 아니라 `옷 / 화장품 / 액세서리 / 선물 / 고급 전자기기`를 묶어 파는 상위 상점 역할을 맡는다.

#### 소비 예시

- 명품 벨트, 시계, 향수 구매
- 고급 선물 구매
- 정장 / 코트 / 구두 세트 구매
- 한정판 전자기기 구매

#### 연결 효과

- 인벤토리 `장비` 또는 `소지품` 탭에 고급 품목 추가
- 평판, 행복도, 외모 보정
- 일부 대화 선택지 / 소개팅 / 상류층 NPC 루트 보정
- 과시형 소비 성향 누적 가능

#### 설계 원칙

- 백화점 소비는 `성능 대비 가성비`는 낮다.
- 대신 `평판`, `외모`, `행복도`, `이벤트 전용 선택지`에 강하다.
- 돈이 많을수록 단순히 수치가 오르는 게 아니라, 다른 계층의 사람처럼 보이게 만든다.

### 4. 옷 소비

핵심 역할: `장착형 성장 소비`

#### 소비 예시

- 작업복
- 단정한 셔츠 / 정장
- 스트릿 패션
- 명품 아우터
- 운동복

#### 연결 효과

- `state.inventory.items`에 의류 아이템 추가
- `inventory.equipped`에 `outfit` 슬롯을 확장해서 장착
- `appearance.attractiveness` 상승
- 면접, 사무직, 연애, 영상/NPC 이벤트 보정
- 작업복은 노동형 직업 패널티 감소

#### 장착 원칙

1. 옷은 구매만으로 끝나지 않고 장착 상태가 있어야 한다.
2. 현재 입은 옷에 따라 대사, 면접, 평판, 외형 연출이 달라질 수 있어야 한다.
3. 일부 직업은 특정 복장일 때만 보너스를 받는다.

### 5. 휴대폰 소비

핵심 역할: `정보 접근력 + 앱 확장 + 사회적 체급`

현재 인벤토리에는 기본 스마트폰이 있으므로, 휴대폰 소비는 매우 자연스럽게 기존 시스템에 붙일 수 있다.

#### 소비 예시

- 보급형 -> 중급형 -> 프리미엄 폰 업그레이드
- 데이터 요금제 업그레이드
- 액정 수리
- 배터리 교체
- 프리미엄 앱 구독

#### 연결 효과

- `inventory.items`, `inventory.equipped.phone` 변경
- Diggle, 뉴스, 증시, 코인 앱에서 더 좋은 정보 제공
- 앱 설치 슬롯 또는 추천 공고 수 증가
- 일부 사회적 연출에서 외모/평판 소폭 보정

#### 유지비

- 통신요금
- 수리비
- 유료 앱 구독

#### 기대 효과 예시

| 폰 등급 | 효과 |
| --- | --- |
| 기본폰 | 기본 앱만 사용 |
| 중급폰 | 검색 힌트 강화, 공고 추가 노출 |
| 프리미엄폰 | 금융/소셜/영상 루트 보정, 외형 과시 효과 |

### 6. 외모꾸미기 소비

핵심 역할: `appearance.attractiveness`를 가장 직접적으로 올리는 소비

현재 코드에는 `state.appearance.attractiveness`와 `surgeryDone` 상태가 이미 있으므로, 외모 소비는 실제 구현 연결점이 분명하다.

#### 소비 예시

- 미용실
- 피부과
- 네일 / 눈썹 / 왁싱
- 향수, 화장품
- 성형외과

#### 연결 효과

- `state.appearance.attractiveness` 직접 상승
- `state.appearance.flags`에 시술/스타일 플래그 기록
- 행복도 상승
- 특정 NPC / 연애 / 영상 / 사기꾼형 루트에서 보너스

#### 리스크

- 성형은 고비용 + 회복 턴 소모
- 과한 외모 소비는 돈을 급격히 태움
- 일부 이벤트는 외모 상승보다 평판 하락 또는 공허감 상승을 동반할 수 있음

### 7. 운동하기 소비

핵심 역할: `체력형 성장 소비`

#### 소비 예시

- 헬스장 일일권
- 월간 회원권
- PT 수업
- 운동복 / 보충제 구매

#### 연결 효과

- `state.stamina` 최대치 상승 또는 회복 효율 상승
- `state.energy` 소모 효율 보정
- 노동형 직업, 배달, 현장직에서 보너스
- 외모 수치 소폭 상승
- 행복도 또는 우울 완화 보정

#### 설계 원칙

- 운동은 당일 시간과 에너지를 쓰지만 장기적으로 더 많은 돈을 벌게 해준다.
- 즉시 수익보다 `다음 턴 생존력`과 `노동 계열 직업 효율`을 높이는 투자다.

## 소비와 스탯 연결표

| 소비 | 직접 상승 | 간접 상승 | 상태 변화 |
| --- | --- | --- | --- |
| 부동산 | 행복도, 평판 | 에너지 회복 | `ownership.home`, `ownership.residence` |
| 자동차 | 평판 | 체력 보존, 이동 효율 | `ownership.vehicle` |
| 백화점 | 행복도, 평판 | 외모 | `inventory.items` |
| 옷 | 외모 | 평판, 면접 성공률 | `inventory.items`, `inventory.equipped.outfit` |
| 휴대폰 | 정보 접근력 | 지능형 투자 효율, 평판 | `inventory.equipped.phone` |
| 외모꾸미기 | 외모 | 행복도, 대인 이벤트 성공률 | `appearance.attractiveness`, `appearance.flags` |
| 운동 | 체력, 에너지 효율 | 외모, 노동 보너스 | 장기 보정값 또는 운동 플래그 |

## 소비와 인벤토리 연결표

| 카테고리 | 인벤토리 처리 |
| --- | --- |
| 옷 | `items`에 저장, `equipped.outfit` 장착 |
| 휴대폰 | `items`에 저장, `equipped.phone` 교체 |
| 백화점 액세서리 | `items`에 저장, 추후 `equipped.accessory` 확장 |
| 화장품 / 향수 | 소모품 또는 외모 버프 아이템 |
| 운동복 / 보충제 | 소지품 또는 장비로 처리 |
| 부동산 문서 | `ownership` 원본 + 인벤토리 문서 카드 파생 |
| 차량 키 | `ownership.vehicle` 원본 + 인벤토리 키 카드 파생 |

## 신규 상태 제안

소비 시스템을 얹을 때는 새 상태를 과하게 만들기보다 기존 상태를 확장하고, 반복 과금만 별도 상태로 분리한다.

```js
state.consumption = {
  recurring: {
    phonePlan: "basic",
    gymMembership: null,
    beautyMembership: null,
    housingBillDueDay: null,
    vehicleBillDueDay: null,
  },
  lastCategory: "",
  spendingLedger: [],
};
```

### 기존 상태 확장 방향

- `state.inventory.equipped`
  `outfit`, `accessory` 슬롯 확장
- `state.appearance.flags`
  헤어, 시술, 고급스타일, 운동습관 같은 플래그 저장
- `state.ownership`
  집/차 등급과 유지비 이벤트 연결

## 구현 우선순위

1. `휴대폰 소비`
   기존 `inventory.equipped.phone`과 바로 연결 가능
2. `옷 소비`
   인벤토리 확장과 면접/평판 보정에 바로 쓰기 좋음
3. `운동하기`
   체력/에너지 루프와 직업 보너스에 바로 연결 가능
4. `외모꾸미기`
   기존 `appearance.attractiveness`와 직접 연결
5. `자동차 소비`
   `ownership.vehicle`과 이동 루프 연결
6. `부동산 소비`
   장기 루프, 회차 시스템과 강하게 연결
7. `백화점 소비`
   사치/평판/연애/NPC 보정용 확장 축

## 구현 순서 제안

```text
옷 / 휴대폰 / 운동
  -> 인벤토리 / 스탯 상승 연결

외모꾸미기
  -> appearance 연결

자동차
  -> ownership.vehicle + 이동 보정

부동산
  -> ownership.home + 회복 / 생활 수준

백화점
  -> 고급 소비 허브 + 상위 NPC / 평판 루트
```

## 직업과의 연결 규칙

소비는 직업과 분리되면 안 된다. 추천 연결은 아래와 같다.

- `옷`
  면접, 사무직, 서비스직, 영상 루트 보정
- `운동`
  노동직, 배달, 현장직 보정
- `휴대폰`
  정보직, 투자, 공고 탐색, 영상 루트 보정
- `자동차`
  배달, 운송, 영업, 장거리 이동 보정
- `부동산`
  회복, 장기 생활 안정, 상위 계층 루트 보정
- `외모꾸미기 / 백화점`
  연애, 인플루언서, 상류층 대화, 사기형 루트 보정

## 최종 목표

플레이어가 같은 100만 원을 쥐고도 전혀 다른 방향으로 인생을 굴릴 수 있어야 한다.

- 집에 넣는 사람
  안정과 회복을 얻음
- 차에 넣는 사람
  속도와 편의를 얻음
- 옷과 외모에 넣는 사람
  평판과 관계 기회를 얻음
- 폰과 정보에 넣는 사람
  더 좋은 정보와 금융/직업 보너스를 얻음
- 운동에 넣는 사람
  생존력과 노동 효율을 얻음

즉 소비는 단순 낭비가 아니라 `플레이 스타일 선택`이 되어야 한다.
