# 시간 시스템 설계

## 핵심 개념

모든 행동은 시간을 소비한다.
시간이 다 떨어지면 하루가 끝나고 다음날로 넘어간다.
플레이어는 하루 안에 할 수 있는 행동이 제한된다.

---

## 하루 시간 구조

```
하루 = 16시간 (오전 8시 ~ 자정)
잠 = 8시간 (자정 이후 자동으로 다음날)
```

---

## 행동별 시간 소비

### 사소한 행동 (0.5 ~ 1시간)

| 행동 | 소비 시간 |
|------|----------|
| 스마트폰 켜기 / 앱 확인 | 0.5시간 |
| 당근마켓 물건 팔기 | 0.5시간 |
| 유튜브 쇼츠 보기 | 1 ~ 2시간 (랜덤) |
| 주식 앱 확인 | 0.5시간 |
| 밥 먹기 | 0.5시간 |

### 외출 행동 (1 ~ 3시간)

| 행동 | 소비 시간 |
|------|----------|
| 밖에 나가기 | 0.5시간 |
| 구인게시판 확인 | 0.5시간 |
| 도서관 이동 | 0.5시간 |
| 도서관 공부 | 2 ~ 3시간 |

### 알바 / 일 (시급 계산)

| 행동 | 소비 시간 | 비고 |
|------|----------|------|
| 단순 노동 알바 | 4 ~ 8시간 | 시급 × 시간 = 수입 |
| 전문직 알바 | 3 ~ 5시간 | 시급 높음 |
| 사기 / 범죄 | 1 ~ 2시간 | 시간 적게 쓰고 돈 많이 버는 구조 |

### 시급 기준 (현실 반영)

| 직종 | 시급 |
|------|------|
| 최저 노동 | 9,860원 |
| 서비스직 | 1만 ~ 1만 5천 원 |
| 전문직 단기 | 3만 ~ 10만 원 |
| 사기 / 불법 | 시간당 수십만 원 (대신 범죄도 상승) |

---

## 시간 흐름 예시

```
오전 8시 시작 (16시간)

스마트폰 켬          → 0.5시간 소비  → 15.5시간 남음
당근마켓 물건 팔기   → 0.5시간 소비  → 15시간 남음
밖에 나감            → 0.5시간 소비  → 14.5시간 남음
알바 선택 (6시간)    → 6시간 소비    → 8.5시간 남음
귀가                 → 0.5시간 소비  → 8시간 남음
유튜브 봄            → 2시간 소비    → 6시간 남음
잠든다               → 다음날
```

---

## 핵심 긴장감

- 알바를 하면 돈은 생기지만 하루가 절반 이상 날아감
- 사소한 행동들이 쌓이면 정작 중요한 일을 못 함
- 도서관 공부 vs 알바 선택이 매일 발생
- 범죄는 시간 대비 돈이 많지만 범죄도 상승

---

## 에너지와 시간의 관계

둘은 별개지만 동시에 소비됨. 어느 하나가 0이면 행동 불가.

| 행동 | 시간 소비 | 에너지 소비 |
|------|----------|------------|
| 스마트폰 / 앱 | 있음 | 없음 |
| 유튜브 | 있음 | 없음 |
| 도서관 공부 | 있음 | 있음 (집중력 소모) |
| 알바 / 노동 | 있음 | 있음 (육체 소모) |
| 범죄 행동 | 있음 | 있음 (긴장으로 소모) |
| 밥 먹기 | 있음 | 에너지 회복 |
| 잠자기 | 없음 (다음날) | 에너지/체력 회복 |

결론: 스마트폰/유튜브는 시간만 날리고 에너지는 안 씀. 알바/공부는 둘 다 소비.
에너지 바닥나면 알바/공부 못 하고 스마트폰만 할 수 있는 상태가 됨.

---

## 원칙

- 시간은 되돌릴 수 없음. 한 번 쓰면 끝
- 시간이 0이 되면 강제로 다음날 (선택 중단)
- 에너지 0이면 노동/공부 불가, 시간만 있어도 소용없음
- 유튜브 같은 낭비 행동은 시간이 랜덤으로 사라지게 설계 (중독성 표현)
- 하루에 알바를 여러 개 할 수 있음 (시간이 허락하면). 단 에너지가 버텨야 함
