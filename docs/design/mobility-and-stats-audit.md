# 이동 / 차량 / 스탯 점검

현재 결론: `이동 시간 차감`, `차량 보유 효과`, `오토바이/차량 기반 배달 게이트`, `통합 스탯 패널`의 1차 구현은 붙었다. 지금부터의 과제는 씨앗을 만드는 게 아니라, 이미 붙은 연결을 더 깊게 만드는 것이다.

## 이번 점검에서 확인한 것

### 1. 이동 시간은 이미 차감된다

현재 이동 시간은 실제로 소비된다. 차감 시점은 `도착`이 아니라 `이동 시작`이다.

#### 현재 동작 방식

- `startWorldTravelToLocation()`에서 이동 분 수를 계산한다.
- 그 즉시 `spendMinorTime(travelMinutes)`를 호출해 시간을 차감한다.
- 이후 씬은 `walk-travel` 또는 `bus-ride`로 바뀌고, `도착한다` 버튼은 위치만 확정한다.

즉 구조는 아래와 같다.

```text
이동 버튼 누름
  -> 이동 시간 차감
  -> 이동 중 연출 씬 진입
  -> 도착 버튼
  -> 위치 확정
```

### 2. 도시 지도 기반 이동 시간 계산도 이미 있다

지도 서비스는 링크 데이터 기준으로 최소 이동 시간을 계산하고, 도착 예정 시각도 보여준다.

#### 현재 계산 요소

- 링크별 분(`minutes`)
- 이동 수단(`walk`, `bus`)
- 최단 경로
- 도착 예정 시각

즉 `도시 지도 -> 소요시간 확인 -> 이동` UX는 이미 기본 뼈대가 있는 상태다.

### 3. 차량은 이제 기본 효과가 붙어 있다

현재 차량 카탈로그와 보유 상태는 있고, 이동 시간 보정에도 실제로 반영된다.

#### 현재 상태

- `ownership.vehicle` 저장됨
- 인벤토리에서 차량 키/차량 카드가 보임
- 이동 시간 계산에서 보유 차량을 읽음
- 일부 직업 해금 조건에 차량이 직접 반영됨

즉 이제는 `차를 가지면 이동과 직업 조건이 달라지는 상태`까지는 들어왔다.

### 4. 오토바이는 1차 구현이 들어갔다

배달 현실성을 살리기 위한 핵심 축으로, 현재는 `used-motorbike`가 중고 매물과 직업 게이트에 연결되어 있다.

#### 아직 남은 문제

- 자전거/오토바이/차의 차이를 더 세밀하게 나눌 필요
- 2종 소형 같은 별도 면허 단계는 아직 미구현
- 장거리 배달과 물류 출근 루프는 더 보강 필요

### 5. 스탯은 통합 패널 1차가 들어갔다

현재 플레이어가 보는 캐릭터 패널은 아래 4묶음으로 확장되어 있다.

- `생활`
- `사회`
- `직업`
- `생활수준`

#### 이미 패널에 노출되는 것

- 체력
- 에너지
- 배고픔
- 행복도
- 평판
- 범죄도
- 외모
- 서비스/현장/사무/학업 준비도
- 자격증
- 현재 거처
- 보유 차량
- 현재 폰 등급
- 현재 복장

즉 이전처럼 내부에만 있던 준비도/자격/장비/생활수준이 이제는 한 화면에서 읽히는 상태까지는 왔다.

## 지금 이대로 가면 안 되는 점

### A. 차량을 아직도 "효과가 약한 장식 자산"으로 두는 것

차량은 이미 기본 효과가 붙었지만, 아래 연결은 더 강해져야 한다.

- 이동 시간
- 직업 가능 여부
- 이동 중 이벤트
- 유지비
- 감가와 되팔기

### B. 배달을 아직도 너무 단순한 카드 직업으로만 두는 것

현재 1차 게이트는 붙었지만, 여전히 아래 확장이 필요하다.

- 자전거/오토바이/차량 루트 세분화
- 2종 소형 / 운전면허 분화
- 장거리 배송과 물류 출근 루프 강화

### C. 상세 스탯을 만들고도 거기서 실제 게이트를 더 안 연결하는 것

`외모`, `준비도`, `장비`, `자격증`, `생활 수준`은 이제 보이기 시작했다. 다음 문제는 이 스탯들이 직업/NPC/소비와 더 강하게 물려야 한다는 점이다.

## 꼭 필요한 연결

### 1. 차량 -> 이동 시간 보정

추천 규칙:

| 보유 차량 | 이동 보정 |
| --- | --- |
| 없음 | 기본 시간 |
| 자전거 | 도보 단거리 `-15%` |
| 오토바이 | 배달/근거리 `-30%`, 일부 직업 해금 |
| 중고차 | 장거리 / 버스 대체 일부 가능 |
| 외제차 | 장거리 효율 + 평판/과시 보정 |

#### 현재 상태

- 이동 시간 보정은 현재 `city-map-service`와 이동 시작 로직에 연결되어 있다.
- UI 문구와 실제 차감 분도 같은 규칙을 따라간다.

### 2. 차량 -> 직업 게이트

배달은 아래처럼 단계화하는 게 맞다.

| 직업 | 최소 조건 |
| --- | --- |
| 도보 배달 | 없음 또는 자전거 권장 |
| 자전거 배달 | 자전거 필수 |
| 오토바이 배달 | 오토바이 + 2종 소형 면허 |
| 장거리 배달 / 운송 | 차량 + 운전면허 |

즉 `배달`을 하나로 두지 말고 탈것 단계에 따라 더 세분화하는 편이 아직도 더 현실적이다.

### 3. 스탯 -> 통합 화면 정리

추천 묶음은 아래와 같다.

#### 생활 스탯

- 체력
- 에너지
- 배고픔
- 행복도

#### 사회 스탯

- 평판
- 범죄도
- 외모

#### 직업 스탯

- 서비스 준비
- 현장 준비
- 사무 준비
- 학업 준비
- 보유 자격증

#### 생활 수준

- 현재 거처
- 보유 차량
- 현재 폰 등급
- 현재 복장

이 통합 화면은 이미 1차로 붙어 있다. 이제 남은 건 이 정보를 더 많은 시스템이 읽게 만드는 것이다.

## 현재 코드 기준 요약

### 이미 되는 것

- 분 단위 시간 차감
- 도시 지도 이동 시간 계산
- 도착 예정 시각 계산
- 차량 보유 상태 저장
- 차량별 이동 시간 단축
- 배달 직업의 차량/장비/자격 일부 게이트
- 통합 상세 스탯 패널

### 아직 안 되는 것

- 유지비/감가를 포함한 차량 자산 루프
- 2종 소형 같은 세부 면허 단계
- 외모/복장/폰 등급이 더 많은 직업/NPC 반응에 직접 연결되는 것
- 통합 스탯 패널에서 바로 액션으로 이어지는 UX

## 구현 우선순위

1. 차량 유지비와 감가 루프 연결
2. 배달 직업을 `도보 / 자전거 / 오토바이 / 차량`으로 더 세분화
3. 2종 소형 / 운전면허 분화
4. 통합 스탯 패널과 직업/NPC/소비 조건을 더 촘촘히 연결
5. 차량 이동 중 이벤트 연결
