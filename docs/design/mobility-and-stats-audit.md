# 이동 / 차량 / 스탯 점검

현재 결론: `이동 시간 차감` 자체는 이미 동작하고 있지만, `차량 보유 효과`, `오토바이 배달 게이트`, `상세 스탯 노출`은 아직 구조만 있고 실제 플레이 체감으로는 거의 붙어 있지 않다. 즉 지금 필요한 건 새 아이디어를 더 얹는 것보다, 이미 있는 이동/보유/스탯 상태를 제대로 연결하는 작업이다.

## 이번 점검에서 확인한 것

### 1. 이동 시간은 이미 차감된다

현재 이동 시간은 실제로 소비된다. 차감 시점은 `도착`이 아니라 `이동 시작`이다.

#### 현재 동작 방식

- `startWorldTravelToLocation()`에서 이동 분 수를 계산한다.
- 그 즉시 `spendMinorTime(travelMinutes)`를 호출해 시간을 차감한다.
- 이후 씬은 `walk-travel` 또는 `bus-ride`로 바뀌고, `도착한다` 버튼은 위치만 확정한다.

즉 구조는 아래와 같다.

```text
이동 버튼 누름
  -> 이동 시간 차감
  -> 이동 중 연출 씬 진입
  -> 도착 버튼
  -> 위치 확정
```

#### 코드 기준

- 시간 차감 함수: `js/logic.js` `spendMinorTime()`
- 이동 시작 함수: `js/logic.js` `startWorldTravelToLocation()`
- 도착 확정 함수: `js/logic.js` `completeWalkTravel()`, `completeBusTravel()`

### 2. 도시 지도 기반 이동 시간 계산도 이미 있다

지도 서비스는 링크 데이터 기준으로 최소 이동 시간을 계산하고, 도착 예정 시각도 보여준다.

#### 현재 계산 요소

- 링크별 분(`minutes`)
- 이동 수단(`walk`, `bus`)
- 최단 경로
- 도착 예정 시각

즉 `도시 지도 -> 소요시간 확인 -> 이동` UX는 이미 기본 뼈대가 있는 상태다.

### 3. 차량은 아직 "보유 표시" 수준이다

현재 차량 카탈로그와 보유 상태는 있다.

- `bicycle`
- `used-car`
- `foreign-car`

하지만 실제 이동 시간 계산이나 직업 가능 여부에 차량이 아직 직접 반영되진 않는다.

#### 현재 상태

- `ownership.vehicle` 저장됨
- 인벤토리에서 차량 키/차량 카드가 보임
- 하지만 이동 시간 계산에서 `ownership.vehicle`을 읽지 않음
- 직업 해금 조건에도 하드 게이트로 거의 안 들어감

즉 지금은 `차가 있어도 그냥 이름만 가진 상태`에 가깝다.

### 4. 오토바이는 아직 없다

배달 현실성을 살리려면 중요한 축인데, 현재 차량 카탈로그에는 오토바이가 없다.

#### 문제

- 배달 직업이 현실보다 너무 쉽게 열린다.
- 자전거/오토바이/차의 차이가 없다.
- 면허와 배달 직업이 강하게 연결되지 않는다.

### 5. 스탯은 아직 충분히 상세하지 않다

현재 플레이어가 체감하는 스탯은 아래 정도다.

- 상단 UI
  - 체력
  - 에너지
  - 배고픔
- 캐릭터 패널
  - 지능
  - 평판
  - 범죄도
  - 행복도
  - 배고픔

#### 이미 상태는 있지만 덜 보이는 것

- `appearance.attractiveness`
- `jobs.careerPrep.service / labor / office / academic`
- `certifications`

즉 내부에 씨앗은 있는데, 플레이어가 한눈에 읽는 `상세 성장 화면`은 아직 아니다.

## 지금 이대로 가면 안 되는 점

### A. 차량을 수집형 자산처럼만 두는 것

차량은 이름만 갖고 있으면 안 된다. 반드시 아래 중 하나 이상을 바꿔야 한다.

- 이동 시간
- 직업 가능 여부
- 이동 중 이벤트
- 유지비

### B. 배달을 그냥 하나의 알바 카드로 두는 것

배달은 생활감이 강한 직업이라 현실 게이트가 꼭 있어야 한다.

지금처럼 두면:

- 자전거도 없이 배달 가능
- 오토바이 사는 이유가 없음
- 2종 소형 / 운전면허 동기가 약함

### C. 상세 스탯을 안 보여주고 시스템만 늘리는 것

`외모`, `준비도`, `장비`, `자격증`, `생활 수준`이 다 숨은 채로 기능만 늘어나면 플레이어는 왜 루트가 열리고 닫히는지 이해하기 어려워진다.

## 꼭 필요한 연결

### 1. 차량 -> 이동 시간 보정

추천 규칙:

| 보유 차량 | 이동 보정 |
| --- | --- |
| 없음 | 기본 시간 |
| 자전거 | 도보 단거리 `-15%` |
| 오토바이 | 배달/근거리 `-30%`, 일부 직업 해금 |
| 중고차 | 장거리 / 버스 대체 일부 가능 |
| 외제차 | 장거리 효율 + 평판/과시 보정 |

#### 구현 원칙

- 이동 시간 보정은 `city-map-service` 또는 `startWorldTravelToLocation()`에서 계산한다.
- UI 문구와 실제 차감 분이 같은 규칙을 따라야 한다.

### 2. 차량 -> 직업 게이트

배달은 아래처럼 단계화하는 게 맞다.

| 직업 | 최소 조건 |
| --- | --- |
| 도보 배달 | 없음 또는 자전거 권장 |
| 자전거 배달 | 자전거 필수 |
| 오토바이 배달 | 오토바이 + 2종 소형 면허 |
| 장거리 배달 / 운송 | 차량 + 운전면허 |

즉 `배달`을 하나로 두지 말고 탈것 단계에 따라 쪼개는 편이 더 현실적이다.

### 3. 스탯 -> 통합 화면 정리

추천 묶음은 아래와 같다.

#### 생활 스탯

- 체력
- 에너지
- 배고픔
- 행복도

#### 사회 스탯

- 평판
- 범죄도
- 외모

#### 직업 스탯

- 서비스 준비
- 현장 준비
- 사무 준비
- 학업 준비
- 보유 자격증

#### 생활 수준

- 현재 거처
- 보유 차량
- 현재 폰 등급
- 현재 복장

이렇게 보이면 플레이어가 "왜 이 직업이 열렸는지", "왜 이 NPC가 다르게 반응하는지"를 읽을 수 있다.

## 현재 코드 기준 요약

### 이미 되는 것

- 분 단위 시간 차감
- 도시 지도 이동 시간 계산
- 도착 예정 시각 계산
- 차량 보유 상태 저장
- 기본 스탯 일부 표시

### 아직 안 되는 것

- 차량별 이동 시간 단축
- 오토바이 보유/면허 게이트
- 차량 보유에 따른 직업 해금
- 통합 상세 스탯 패널
- 외모/준비도/자격증/생활수준의 한 화면 통합 표시

## 구현 우선순위

1. `오토바이` 자산 추가
2. `ownership.vehicle` 기반 이동 시간 보정
3. 배달 직업에 `자전거 / 오토바이 / 면허` 조건 추가
4. 캐릭터 패널을 `생활 / 사회 / 직업 / 생활수준` 묶음으로 확장
5. 차량 유지비와 이동 이벤트 연결

## 구현 메모

### 이동 계산은 여기서 잡는 게 맞다

- `js/systems/world/city-map-service.js`
- `js/logic.js` `startWorldTravelToLocation()`

### 차량 상태는 여기서 가져오면 된다

- `js/systems/economy/inventory-service.js`
  - `ownership.vehicle`
  - `getOwnedVehicleDefinition()`
  - `setOwnedVehicle()`

### 스탯 표시 확장은 여기서 시작하면 된다

- `js/ui.js`
  - 캐릭터 패널 렌더링
- `js/logic.js`
  - 기본 스탯 루트 상태
- `js/systems/presentation/presentation-service.js`
  - 외모 수치
- `js/apps/jobs/jobs-app-ui.js`
  - 직업 준비도 / 자격증 요약

## 한 줄 요약

이동 시간은 이미 잘 깎이고 있다. 지금 비어 있는 건 `차량 효과`, `오토바이 배달 현실성`, `상세 스탯 가시화`다.
