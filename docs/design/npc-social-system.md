# NPC / 연애 / 반응 시스템

현재 결론: NPC는 배경 장식이 아니라 `직업`, `소비`, `연애`, `돌발 이벤트`, `루트 해금`을 연결하는 핵심 장치여야 한다. 먼저 `돌아다녀도 NPC가 안 보이는 문제`를 해결하고, 그 다음 `각 NPC가 무슨 역할인지`, `왜 또 만나야 하는지`, `플레이어 외모/옷/성형/행복도에 왜 반응하는지`를 시스템으로 정리해야 한다.

## 현재 보이는 문제

- 돌아다녀도 NPC가 거의 안 보이거나, 보이더라도 존재감이 약하다.
- NPC를 만나도 "이 사람이 왜 중요한지"를 플레이어가 알기 어렵다.
- 랜덤 돌발 이벤트와 연결되지 않아 도시가 밋밋하게 느껴진다.
- 성형, 외모, 옷, 소비 변화가 일부 대사에만 걸려 있고 전반적 반응 시스템으로 보이진 않는다.
- 연애 / 여자친구 / 호감 / 소개팅 / 관계 유지가 아직 하나의 루프로 정리되지 않았다.

## 핵심 목표

1. 주요 장소마다 최소 1명의 의미 있는 NPC가 보여야 한다.
2. NPC마다 `역할`, `보상`, `위험`, `관계 방향`이 분명해야 한다.
3. 플레이어 상태 변화에 따라 NPC 반응이 달라져야 한다.
4. 연애는 행복도를 크게 바꾸는 대표 루트여야 한다.
5. 랜덤 돌발 이벤트가 NPC를 통해 들어와야 한다.

## NPC 5종 분류

### 1. 고정 시설 NPC

- 편의점 점원
- 자동차매장 영업사원
- 부동산 공인중개사
- 백화점 판매사원
- 증권사 직원

역할: 기능 안내, 거래 시작, 시설 전용 이벤트 제공

### 2. 생활형 거리 NPC

- 동네 아주머니
- 퇴근길 직장인
- 학생
- 버스정류장 이용객

역할: 도시 분위기, 소문, 소소한 인연, 생활형 돌발 이벤트

### 3. 루트 해금 NPC

- 사업 제안자
- 범죄 브로커
- 고액 과외 연결자
- 투자 정보상

역할: 새 직업/사업/범죄/투자 루트 진입

### 4. 연애 가능 NPC

- 편의점 점원
- 도서관 이용자
- 백화점 VIP 고객
- 헬스장 트레이너 / 회원

역할: 호감, 데이트, 이별, 행복도, 결혼 루트

### 5. 사건 유발 NPC

- 사기꾼
- 사이비 포섭자
- 중고차 브로커
- 내부자 정보 제공자

역할: 위험, 유혹, 파탄, 돌발 이벤트

## NPC는 반드시 역할을 가져야 한다

모든 NPC는 아래 네 줄 요약이 있어야 한다.

```text
이 NPC는 어디서 만나는가
이 NPC는 무엇을 주는가
이 NPC는 어떤 리스크를 가지는가
이 NPC와 친해지면 무엇이 열리는가
```

예:

- 편의점 점원
  - 위치: 편의점
  - 기능: 생활 대화, 소소한 행복도 회복, 연애 루트
  - 리스크: 거절/실수 시 어색함, 호감도 하락
  - 보상: 연락처 획득, 데이트 루트, 행복도 상승

## 관계 상태 구조

현재 코드에는 이미 `affinity`, `attraction`, `annoyance`, `flags`가 있으므로, 이 축을 중심으로 확장하는 것이 가장 안전하다.

### 현재 확장 방향

```js
state.npcs.relations[npcId] = {
  met: false,
  meetings: 0,
  lastSeenDay: 0,
  affinity: 0,
  attraction: 0,
  annoyance: 0,
  trust: 0,
  romanceStage: "none",
  roleKnown: false,
  flags: {},
};
```

### 필드 의미

- `affinity`
  전반적 호감
- `attraction`
  외모적 / 이성적 끌림
- `annoyance`
  짜증, 불쾌감, 부담감
- `trust`
  돈, 소개, 비밀, 사업 제안을 맡길 수 있는 신뢰도
- `romanceStage`
  `none -> interest -> contact -> dating -> serious -> broken`
- `roleKnown`
  이 NPC가 어떤 기능/루트를 가진 사람인지 플레이어가 파악했는지 여부

## NPC 반응 레이어

NPC는 한 가지 값만 보고 반응하면 안 된다. 아래 레이어를 섞어 반응해야 한다.

### 1. 외모 / 성형 반응

이미 코드에는 `appearance.attractiveness`, `surgeryDone`, `appearance.flags`가 있다. 이걸 전역 반응 시스템으로 승격한다.

- 외모 수치 상승 -> 첫인상 보너스
- 성형 완료 -> 일부 NPC 관심 증가, 일부 NPC 거부감도 가능
- 스타일 플래그 -> 옷 성향별 반응 분기

### 2. 옷 / 장비 반응

- 정장 -> 사무직, 중개사, 증권사 직원 반응 상승
- 운동복 -> 헬스장/NPC 반응 상승
- 명품 / 백화점 룩 -> 상류층, 과시형 NPC 반응 상승
- 지저분한 복장 -> 면접, 연애, 서비스직 반응 하락

### 3. 생활 수준 반응

- 집
- 차
- 휴대폰 등급
- 최근 소비 패턴

즉 NPC는 플레이어가 단지 돈이 많은지가 아니라, `무슨 사람처럼 보이는지`에 반응해야 한다.

### 4. 행복도 반응

- 행복도 높음 -> 여유 있고 부드러운 대화
- 행복도 낮음 -> 예민한 선택지, 자포자기형 반응
- 우울 상태 -> 일부 연애 선택지 봉쇄 또는 불리한 대화

## 반응 판정식 예시

```text
NPC 첫 반응 점수
  = 기본 태도
  + affinity
  + attraction
  + 외모 보정
  + 복장 보정
  + 생활 수준 보정
  + 행복도 보정
  - annoyance
```

## 연애 시스템 초안

연애는 `행복도 회복 장치 + 장기 분기 루트 + 소비 동기`를 동시에 맡는다.

### 기본 흐름

```text
처음 만남
  -> 반복 접촉
  -> 연락처 교환
  -> 데이트 제안
  -> 사귀기
  -> 관계 유지 / 이별 / 결혼
```

### 연애 시작 조건 예시

- 일정 이상 `affinity`
- 일정 이상 `attraction`
- 일정 이하 `annoyance`
- 외모 / 복장 / 행복도 / 평판 최소치
- 해당 NPC 전용 플래그 충족

### 연애 효과

- 행복도 대폭 상승
- 특정 소비(선물, 데이트, 외모관리)에 보상 발생
- 일부 직업 루트 / 소비 루트 / 엔딩 분기 변화

### 연애 유지 비용

- 데이트 비용
- 선물 비용
- 시간 투자
- 행복도/관심 관리 실패 시 이별

## 여자친구 시스템 원칙

1. 단순히 호감도 숫자만 쌓는 구조로 두지 않는다.
2. 연락, 데이트, 선물, 스타일, 생활 수준이 모두 영향 줘야 한다.
3. 연애는 행복도 상승이 크지만 유지비도 큰 선택이어야 한다.
4. 연애 성공 시 삶이 달라진 느낌이 들어야 한다.

## 랜덤 돌발 이벤트

NPC는 랜덤 이벤트의 입구가 되어야 한다.

### 생활형 돌발 이벤트

- 길에서 우연히 재회
- 연락처를 갑자기 받음
- 버스에서 자리 시비
- 동네 소문 전달

### 소비형 돌발 이벤트

- 백화점 VIP 초대
- 중고차 사기 제안
- 급매 아파트 정보 유출
- 증권 브로커의 내부 정보 유혹

### 연애형 돌발 이벤트

- 우연한 재회 후 데이트 제안
- 연락 씹힘 / 오해 / 질투
- 스타일 바뀐 후 반응 급변
- 성형 후 재접촉 대사 변화

### 범죄형 돌발 이벤트

- 사채 브로커 접근
- 사이비 포섭
- 정보 장사 제안
- 범죄 루트 동업 제안

## NPC가 밋밋하지 않으려면

NPC는 항상 아래 중 하나를 수행해야 한다.

- 정보를 준다
- 돈을 벌게 하거나 잃게 한다
- 감정을 바꾼다
- 루트를 연다
- 사건을 일으킨다

아무것도 하지 않는 NPC는 장식 NPC로만 제한하고, 대화 가능한 NPC는 반드시 기능이 있어야 한다.

## UI 원칙

### 화면에서 바로 보여야 하는 것

- NPC 이름
- 역할 태그
- 현재 태도 톤
- 왜 중요한지 짧은 문구

예:

```text
편의점 점원
호감 가능 / 생활 NPC
요즘은 먼저 말을 걸면 잘 받아준다.
```

### 관계 화면에서 보여야 하는 것

- 만난 횟수
- 친밀도 단계
- 연락 가능 여부
- 데이트 가능 여부
- 최근 반응 메모

## 구현 우선순위

1. `NPC 스폰 안정화`
   돌아다닐 때 주요 장소에 NPC가 실제로 보이게 하는 것부터
2. `NPC 역할 가시화`
   이 사람이 왜 중요한지 태그/문구로 보이게 만들기
3. `랜덤 돌발 이벤트`
   도시가 살아 있는 느낌 만들기
4. `외모 / 옷 / 성형 반응`
   현재 `appearance`와 `presentation`을 전역화
5. `연애 / 여자친구 시스템`
   행복도와 장기 루트 연결

## 구현 순서 제안

```text
NPC 보이기
  -> NPC 역할 보이기
  -> NPC 랜덤 사건
  -> NPC 반응 다양화
  -> 연애 / 데이트
  -> 결혼 / 파탄 / 장기 분기
```

## 기존 코드와 맞는 연결점

- `state.appearance.attractiveness`
- `state.appearance.surgeryDone`
- `state.appearance.flags`
- `state.npcs.relations[npcId]`
- `state.happiness`
- `getNpcPresentation()`
- `patchNpcRelation()`

즉 기초 상태는 이미 있고, 지금 필요한 건 `전역 규칙`과 `콘텐츠 양`이다.

## 최종 목표

도시를 돌아다니다가 누구를 만나느냐, 어떻게 보이느냐, 무슨 옷을 입었느냐, 성형을 했느냐, 행복도가 어떤 상태냐에 따라 같은 장소가 전혀 다르게 느껴져야 한다.

- 잘 꾸민 날엔 호감과 기회가 열린다.
- 초라한 날엔 무시당하거나 다른 선택지가 뜬다.
- 연애는 돈과 시간을 잡아먹지만 삶을 바꾼다.
- 어떤 NPC는 구원자가 되고, 어떤 NPC는 파멸의 입구가 된다.
