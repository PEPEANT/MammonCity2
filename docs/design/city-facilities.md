# 도시 시설 확장

현재 결론: `자동차매장`, `부동산 공인중개사`, `백화점`, `증권거래소`는 단순 상점이 아니라 플레이어의 계층 상승과 플레이 스타일을 바꾸는 핵심 장소로 들어가야 한다. 이 시설들은 지도 위의 실제 장소여야 하고, 소비/소유/투자/NPC 루트와 동시에 연결돼야 한다.

## 목적

- 소비를 앱 버튼이 아니라 `도시의 실제 장소`로 확장한다.
- 플레이어가 번 돈을 어디에 쓰느냐에 따라 도시 안에서 움직이는 동선이 달라지게 만든다.
- 부동산, 차량, 패션, 금융, 사업 확장을 각각 다른 시설 허브로 분리한다.
- 시설마다 전용 NPC, 전용 이벤트, 전용 위험을 붙여 도시가 살아 있는 느낌을 만든다.

## 신규 시설 목록

### 1. 자동차매장

핵심 역할: `차량 구매 / 업그레이드 / 이동 효율 / 배달·운송 루트 강화`

#### 제공 기능

- 자전거, 중고차, 외제차 구매
- 시승
- 할부 / 대출 제안
- 차량 정비 / 보험 갱신
- 차량 판매 및 교체

#### 연결 시스템

- `state.ownership.vehicle`
- 이동 시간 보정
- 배달 / 운송 / 영업 직군 보너스
- 과시 소비 및 평판 보정

#### 시설 전용 NPC 예시

- 영업사원
- 정비사
- 사채성 할부 브로커

### 2. 부동산 공인중개사

핵심 역할: `거처 이동 / 임대 / 매매 / 전세사기 / 생활 수준 변화`

#### 제공 기능

- 고시원 / 원룸 / 오피스텔 / 아파트 매물 열람
- 월세 / 전세 / 매매 선택
- 보증금, 월세, 관리비 확인
- 계약 체결 및 파기
- 급매 / 경매 / 압류 / 사기 이벤트

#### 연결 시스템

- `state.ownership.home`
- `state.ownership.residence`
- 월세 / 관리비 / 회복 효율
- 전세사기 / 보증금 손실 / 계층 상승 이벤트

#### 시설 전용 NPC 예시

- 공인중개사
- 집주인
- 사기성 브로커
- 급매 투자꾼

### 3. 백화점

핵심 역할: `상위 소비 허브 + 외모 / 평판 / 선물 / 사치 소비`

#### 제공 기능

- 의류, 정장, 향수, 액세서리 구매
- 선물 구매
- 한정판 / 명품 매장
- VIP 등급 및 단골 혜택
- 이벤트성 세일 / 시즌 한정 판매

#### 연결 시스템

- `state.inventory.items`
- `inventory.equipped.outfit`, `inventory.equipped.accessory` 확장
- `state.appearance.attractiveness`
- `state.happiness`
- 면접, 연애, 상류층 NPC 반응

#### 시설 전용 NPC 예시

- 판매사원
- VIP 고객
- 명품관 직원
- 카드사 제휴 영업

### 4. 증권거래소

핵심 역할: `개인 투자 -> 대주주 진입 -> 기업 인수 / 매각`

지금 있는 증시/코인 앱보다 한 단계 위의 장소다. 스마트폰 앱이 `개인 투자자 UI`라면, 증권거래소는 `법인 / 대주주 / 인수전`으로 올라가는 상위 허브다.

#### 제공 기능

- 일반 주식 매수/매도
- 블록딜 제안
- 기업 지분 인수
- 적대적 인수 시도
- 보유 회사 매각
- 상장폐지 / 우회상장 / 작전주 이벤트

#### 연결 시스템

- `state.business`
- `state.money`
- 주식 시장 시스템
- 범죄도 / 평판 / 지능 조건
- 특정 기업 루트 / 오너 엔딩

#### 전개 단계

```text
개인 투자
  -> 고위험 종목 집중
  -> 특정 회사 지분 누적
  -> 이사회 / 경영권 이벤트
  -> 회사 인수 또는 매각
```

#### 시설 전용 NPC 예시

- 증권사 직원
- 기업 사냥꾼
- 작전 세력 브로커
- M&A 자문역

## 지도 배치 원칙

이 시설들은 휴대폰 앱으로만 존재하면 안 된다. 지도 위의 실제 장소로 보이고, 방문해야 하는 이유가 있어야 한다.

### 추천 배치

- 자동차매장
  상업지구 외곽 / 도로 접근성이 좋은 구역
- 부동산 공인중개사
  역 앞 / 주거지구 경계
- 백화점
  다운타운 중심 상권
- 증권거래소
  금융지구 / 고층 빌딩 지대

### 위치 연동 원칙

1. 앱으로 사전 정보는 볼 수 있다.
2. 하지만 최종 계약, 시승, 대형 거래, 특수 제안은 직접 방문이 필요하다.
3. 시설에 따라 특정 NPC와만 만날 수 있는 독점 이벤트가 있어야 한다.

## 소비 / 자산 / 사업 연결표

| 시설 | 핵심 결과 | 연결 상태 |
| --- | --- | --- |
| 자동차매장 | 차량 구매, 이동 효율 상승 | `ownership.vehicle` |
| 부동산 공인중개사 | 거처 변경, 회복량 변화 | `ownership.home`, `ownership.residence` |
| 백화점 | 옷/선물/명품 구매 | `inventory`, `appearance`, `happiness` |
| 증권거래소 | 지분 매집, 회사 인수/매각 | `business`, `money`, 주식 상태 |

## 회사 인수 / 매각 시스템 초안

증권거래소는 단순 주식판이 아니라, 후반부에 회사를 사고파는 `기업 지배` 시스템으로 확장한다.

### 최소 루프

```text
회사를 발견한다
  -> 지분을 조금씩 산다
  -> 특정 지분율 이상 확보
  -> 인수 이벤트 발생
  -> 인수 성공 / 실패
  -> 회사를 운영하거나 비싸게 되판다
```

### 인수 조건 예시

- 충분한 현금 또는 대출 여력
- 지능 최소치
- 평판 또는 범죄도 조건
- 특정 회사 관련 뉴스 / 사건 발생

### 결과 예시

- 성공
  - 배당 수익
  - 사업 확장
  - 오너/재벌 루트 진입
- 실패
  - 주가 폭락
  - 빚 증가
  - 평판 하락 또는 범죄도 상승

## 시설별 돌발 이벤트

### 자동차매장

- 사고 이력 숨긴 중고차
- 시승 중 사고
- 외제차 과시 소비 후 유지비 압박

### 부동산 공인중개사

- 허위 매물
- 전세사기
- 급매 대박 기회

### 백화점

- 충동구매
- 한정판 경쟁
- VIP 초대 행사

### 증권거래소

- 내부자 정보 유혹
- 상장폐지 폭탄
- 적대적 인수전 개시

## 구현 우선순위

1. `백화점`
   현재 `소비 / 인벤토리 / 외모` 시스템과 가장 빨리 연결 가능
2. `부동산 공인중개사`
   `ownership.home`, `residence`와 직접 연결 가능
3. `자동차매장`
   `ownership.vehicle`과 이동 루프 연결
4. `증권거래소`
   기존 증시 시스템 위에 상위 레이어로 확장
