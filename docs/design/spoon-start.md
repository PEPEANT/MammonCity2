# 수저 시작 구조
결론: 수저 추첨은 이제 `초기 자산 차이`만이 아니라 `스폰 위치 -> 집앞 -> 로비 -> 방 -> 바깥`으로 이어지는 시작 생활 루프의 기준점으로 본다. 다음 개발은 이 시작 구조를 기준으로 초반 생존, 직장 준비, 저장/엔딩 반영 순서로 붙인다.

## 목적

- 이미 만들어진 수저별 시작 화면과 집 구조를 다음 개발의 기준점으로 고정한다.
- 수저 로직이 `자산`, `배경`, `이동`, `저장`, `초반 밸런스`에 흩어지지 않게 묶는다.
- 오늘 이후 작업이 `새 연출 추가`인지 `실제 플레이 차이 추가`인지 헷갈리지 않게 순서를 정한다.

## 현재 기준점

### 현재 구현 기준으로 인정할 것

- `js/systems/progression/spoon-start-service.js`
  - 수저 확률
  - 손현금
  - 계좌 잔고
  - 시작 자산
  - 시작 차량
  - 시작 행복도
- 시작 연출은 `수저 추첨 -> 패키지 적용 -> day01 진입` 흐름으로 유지한다.
- 수저별 시작 하우스 자산 폴더가 이미 있다.
  - `assets/days/dirt_spoon`
  - `assets/days/silver_spoon`
  - `assets/days/golden_spoon`
- 다음 작업의 기준선으로 아래 구조를 본다.
  - 수저별 스폰 위치
  - 수저별 집앞 배경
  - 로비 이동
  - 방 이동
  - 방에서 바깥으로 나가는 흐름

### 지금 코드에서 보이는 구조적 주의점

- `world` 런타임은 아직 하루의 기본 집 위치를 `homeLocationId` 하나로 보는 축이 강하다.
- 방/배경/이동 처리는 `ui.js`, `logic.js`, `layout.css`, `world.js`에 나뉘어 있다.
- 따라서 다음 작업은 새 연출을 더 넣기 전에 `시작 집 경로 설정`을 먼저 한 군데로 모아야 한다.

## 비주얼 티어 매핑

현재 수저는 5단계지만 집 비주얼 세트는 3종이다. 그래서 먼저 아래 매핑을 기준으로 굴리는 것이 안전하다.

| 수저 | 비주얼 세트 |
| --- | --- |
| `gold` | `golden_spoon` |
| `silver` | `silver_spoon` |
| `bronze` | `silver_spoon` |
| `steel` | `dirt_spoon` |
| `dirt` | `dirt_spoon` |

이 매핑은 임시가 아니라 현재 개발 기준의 공식 기본값으로 둔다.  
나중에 `bronze`, `steel` 전용 세트가 생기면 그때만 분기 수를 늘린다.

## 목표 루프

1. 수저 추첨 결과 확정
2. 자산 패키지 적용
3. 수저별 스폰 화면 진입
4. 집앞 -> 로비 -> 방 이동
5. 방 -> 집밖 -> 바깥 월드 진입
6. 초반 생존/준비 루프 시작
7. 저장/이어하기/엔딩에서 수저와 시작 집 구조 유지

## 개발 순서

### 0. 시작 집 설정을 한 군데로 모으기

가장 먼저 `수저 -> 시작 집 경로` 설정 객체를 만든다.

권장 필드:

```js
{
  visualTierId: "golden_spoon",
  spawnBackground: "...",
  homeExteriorBackground: "...",
  lobbyBackground: "...",
  roomBackground: "...",
  homeOutsideLocationId: "...",
  lobbySceneId: "...",
  roomSceneId: "...",
}
```

이 설정을 기준으로 이후 배경, 씬 전환, 저장 복원이 모두 같은 값을 보게 한다.

### 1. 씬 전환 정리

먼저 고칠 것은 `이동 로직`이다.

- 프롤로그 종료 후 어디서 시작하는지
- 집앞에서 로비로 들어가는지
- 로비에서 방으로 들어가는지
- 방에서 다시 로비/바깥으로 나가는지

이걸 수저별 `if` 분기로 늘리지 말고, 위 설정 객체를 읽는 공통 함수로 바꾼다.

### 2. 초반 생활 차이 붙이기

시작 집이 잡히면 그다음은 실제 플레이 차이를 붙인다.

- `dirt`
  - 초반 현금 압박
  - 값싼 이동/생존 루프 우선 안내
- `silver`
  - 중간형 출발
  - 준비 루프와 생존 루프를 모두 열어둠
- `gold`
  - 여유 자금
  - 직장 준비와 자산 활용 루프를 빨리 연결

중요한 점은 `문구 차이`보다 `첫 2~3턴 의사결정 차이`를 만드는 것이다.

### 3. 거처 시스템과 연결

지금 시작 집 구조는 나중에 부동산/자취/이사로 교체될 예정이므로, 시작 집도 `임시 연출`이 아니라 거처 시스템의 첫 값으로 들어가야 한다.

- 시작 수저에 따라 초기 거처 키 부여
- 현재 머무는 집/방 정보와 연결
- 나중에 원룸 구매나 이사로 자연스럽게 교체 가능하게 유지

### 4. 저장/이어하기 연결

이 단계에서 반드시 붙여야 한다.

- 수저 티어
- 비주얼 세트 키
- 현재 시작 집 내부 위치
- 방/로비/집앞 중 어디에 있는지

이 값이 저장돼야 이어하기에서 시작 집 연출이 깨지지 않는다.

### 5. 엔딩/랭킹 반영

엔딩에는 최소한 아래 정보가 반영되면 된다.

- 출신 수저
- 최종 직업
- 최종 자산

시작 집 배경 자체를 엔딩에 다시 보여줄 필요는 없지만, `어디서 출발했는가`는 랭킹 문구나 엔딩 태그에서 기억되게 두는 편이 좋다.

### 6. 이후 확장

위가 안정화된 뒤에만 아래를 붙인다.

- 가족 지원 이벤트
- 수저별 부모/NPC 반응
- 수저별 초반 대사
- 수저별 직장 준비 압박
- 수저별 소비 습관/행복도 차이

## 오늘 기준 권장 작업 순서

오늘 바로 손대기 좋은 순서는 아래다.

1. `수저 -> 시작 집 설정 객체` 만들기
2. `방/로비/집앞` 전환을 그 설정으로 읽게 바꾸기
3. 저장 데이터에 `visualTierId`와 현재 시작 집 위치 넣기
4. 그 다음에만 초반 밸런스 차이 추가

## 오늘 안 할 것

- 수저별 NPC 전체 분기
- 수저별 전용 도시 맵
- 가족 지원 시스템 전체
- 엔딩 연출 전체 리뉴얼
- 5단계 수저 모두의 전용 하우스 세트 제작

## 체크리스트

### 구현 체크

- 새 게임 시작 시 수저별로 맞는 스폰 배경이 나온다.
- 집앞 -> 로비 -> 방 이동이 수저별 배경으로 유지된다.
- 방에서 바깥으로 나가면 시작 집의 외부 위치와 이어진다.
- 이어하기 후에도 같은 집 구조와 위치가 복원된다.

### 밸런스 체크

- `gold`, `silver`, `dirt` 시작 3판이 서로 다른 첫 선택을 만들 만큼 차이가 난다.
- 차이는 있지만 첫 1턴에서 바로 게임이 망하지는 않는다.
- 직장 준비 루프와 생존 루프가 수저에 따라 우선순위만 달라지고, 한쪽이 완전히 막히지는 않는다.
