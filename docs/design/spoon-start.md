# 수저 시작 추첨
새 게임 시작 시 1회만 수저 등급을 추첨해 초반 3일의 출발 조건을 바꾼다.

## 목적

- 시작 화면에 강한 첫인상을 만든다.
- 같은 1일차라도 출발선의 차이를 체감하게 한다.
- 다만 수저 결과만으로 런 전체가 결정되지는 않게 유지한다.

## 현재 결정

- 등급은 `금수저 / 은수저 / 동수저 / 쇠수저 / 흙수저` 5단계만 쓴다.
- `다이아 수저`는 제외한다.
- 문구는 짧고 바로 이해되게 간다.
- 추첨은 `새 게임 시작 시 1회만` 실행한다.
- `이어하기`에서는 다시 뽑지 않는다.
- 연출은 무거운 3D보다 `3D처럼 보이는 추첨 연출`을 우선한다.
- 시작 차이는 `현금 한 줄`보다 `출생 자산 패키지` 중심으로 옮긴다.

## 추천 확률

게임용 확률은 사회 서열 설명과 별개로 겹치지 않게 다시 자른다.

| 등급 | 확률 | 게임용 한 줄 설명 |
| --- | ---: | --- |
| 금수저 | 1% | 좋은 교육과 안전망. 시작부터 실패를 만회할 여유가 있다. |
| 은수저 | 4% | 부족함 없이 자랐다. 웬만한 선택은 다시 고를 수 있다. |
| 동수저 | 15% | 평균보다 낫지만 방심하면 금방 흔들린다. |
| 쇠수저 | 30% | 크게 부족하진 않지만 버퍼가 얇다. |
| 흙수저 | 50% | 시작부터 생존 압박이 크다. 작은 실수도 크게 돌아온다. |

## 추천 시작값 초안

현재는 `초기 현금`만 올리는 것보다 `현금 + 계좌 잔고 + starterAssetIds` 구조가 더 현실적이다. 상세 방향은 아래 별도 문서에 정리한다.

- 상세 규칙 문서: `docs/design/wealth-and-cashflow.md`

초기 현금만 따로 볼 경우의 숫자는 보조값으로만 다룬다.

| 등급 | 시작 현금 | 시작 보정 | 메모 |
| --- | ---: | --- | --- |
| 금수저 | 120,000원 | 회복 아이템 2개, 하루 1회 가족 안전망 | 초반 1~2번 실수를 흡수할 수 있어야 한다. |
| 은수저 | 70,000원 | 회복 아이템 1개, 교통/식비 부담 완화 | 금수저보다 여유는 적지만 출발은 안정적이다. |
| 동수저 | 30,000원 | 소형 소비 아이템 1개 | 현재 기본 루프보다 약간 유리한 정도로 둔다. |
| 쇠수저 | 10,000원 | 보정 없음 | 사실상 현재 기본 시작점에 가깝다. |
| 흙수저 | 0원 | day 1~2 강제 지출 또는 소액 빚 이벤트 | 불리하지만 게임이 망하지는 않아야 한다. |

## 수저별 짧은 결과 문구

- 금수저: 좋은 교육과 안전망. 실패해도 다시 고를 수 있다.
- 은수저: 부족함 없는 성장. 급할 때 버틸 여유가 있다.
- 동수저: 무난한 출발. 잘 굴리면 확실히 올라갈 수 있다.
- 쇠수저: 빠듯하지만 버틸 수는 있다. 실수 관리가 중요하다.
- 흙수저: 시작부터 생존전이다. 오늘 하루가 바로 돈이다.

## 시스템 흐름

1. 닉네임 입력 후 `출생 추첨` 단계로 진입한다.
2. 짧은 추첨 연출 후 수저 결과를 공개한다.
3. 결과에 맞는 `초기 현금 / 아이템 / 압박 이벤트`를 현재 런에 적용한다.
4. 적용 후 day01 인트로 또는 시작 화면 본문 카피를 수저 등급에 맞게 바꾼다.
5. 세이브에는 현재 런의 수저 결과를 저장한다.
6. `새로 시작`을 누르면 다시 추첨한다.

## 추천 상태 필드

```js
state.startTier = {
  id: "dirt",
  label: "흙수저",
  applied: false,
  initialCash: 0,
  debtPressure: 30000,
  starterItems: [],
  introCopy: "시작부터 생존전이다.",
};
```

필요하면 별도 필드를 추가한다.

- `state.startTierId`
- `state.startTierApplied`
- `state.startTierDebtDueDay`
- `state.startTierSafetyNetUsed`

## 구현 메모

- 첫 구현은 `통계/히스토리` 없이 단발성 추첨만 넣는다.
- 반복 뽑기 UI는 넣지 않는다.
- 진짜 3D 엔진 대신 `CSS transform`, `blur`, `scale`, `canvas` 정도로 연출한다.
- 흙수저도 역전 가능해야 하므로, `시작부터 빚 100만` 같은 과한 수치는 피한다.
- 가족 안전망이 아직 시스템에 없으면 `1회 무료 식비/교통비 면제` 같은 단순 효과로 대체한다.

## 리스크

- 수저 격차가 너무 크면 플레이보다 출생 결과가 더 중요해진다.
- 텍스트가 길어지면 시작 화면 템포가 죽는다.
- 현실 계급 서술을 너무 직접적으로 가져오면 게임 문구가 거칠어질 수 있다.
- 부채 시스템이 미구현 상태라면 흙수저 효과를 이벤트 형태로 우회해야 한다.
