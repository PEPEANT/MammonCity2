# 행복도 시스템

결론: 현재 프로토타입은 `행복도`를 단일 수치로 두고, `우울증`은 별도 숫자 대신 `우울한 상태`라는 파생 상태로 처리하는 쪽이 가장 안전하다.

## 목적

- 돈, 범죄, 평판만으로는 설명되지 않는 삶의 질 축을 추가한다.
- 연애, 결혼, 친한 NPC, 고립 같은 관계 이벤트를 숫자와 엔딩에 연결한다.
- `돈이 많아도 불행할 수 있다`는 배금시의 풍자를 랭킹과 결산에 반영한다.

## 현재 설계 결론

### 1차 구현

- `행복도`: `0 ~ 100`
- 하루가 바뀔 때 기본 감소
- 연애 성공, 결혼, 친한 NPC 접촉, 좋은 소비/휴식으로 상승
- 고립, 실연, 수면 부족, 장기 빈곤으로 하락
- `행복도 25 ~ 30 이하`에서는 별도 숫자 대신 `우울한 상태` 배지를 띄운다

### 이번 단계에서 하지 않는 것

- `우울증 0 ~ 100` 별도 수치
- 정신 상태를 두 개 수치로 동시에 관리하는 구조
- 의료/약물 시스템과 직접 연결된 임상적 우울증 모델

## 왜 단일 수치로 가는가

- 심리학적으로 행복과 우울은 완전한 반비례가 아니지만, 현재 프로토타입 범위에서는 두 숫자를 동시에 관리하면 구현 복잡도만 커진다.
- 현재 캐릭터 패널, 엔딩 요약, 랭킹 UI는 단일 수치 추가가 가장 자연스럽다.
- `행복도 낮음 -> 우울한 상태 발동`은 플레이 감각이 직관적이고, 나중에 별도 정신 상태 축으로 확장하기도 쉽다.

즉 지금은:

```text
행복도 = 현재 삶의 질 수치
우울한 상태 = 행복도가 너무 낮을 때 붙는 상태이상
```

## 핵심 루프

```text
하루 시작 / 하루 종료
  -> 행복도 기본 감소
  -> 연애 / 결혼 / 친구 / 휴식 / 소비 / 고립 / 실연 결과 반영
  -> 행복도 임계값 확인
  -> 우울한 상태 발동 여부 갱신
  -> 캐릭터 패널, NPC 반응, 결산 요약에 반영
```

## 상태 구조 제안

```js
state.happiness = {
  value: 45,
  status: "steady", // steady | low | depressed
  dailyDecay: 5,
  lastModifiedDay: 1,
};
```

### 상태 의미

- `value`
  현재 행복도 수치
- `status`
  UI와 페널티 판정용 파생 상태
- `dailyDecay`
  기본 일일 감소량
- `lastModifiedDay`
  하루 중복 감소 방지용

## 추천 판정 규칙

### 기본 범위

- `70 이상`
  안정적, 좋은 반응이 붙기 쉬움
- `30 ~ 69`
  보통
- `29 이하`
  `우울한 상태`

### 우울한 상태 효과

- 에너지 회복량 감소
- 일부 NPC 대사 톤 변화
- 방치 시 관계 이벤트에서 불리한 선택지가 열리거나, 좋은 선택지가 막힐 수 있음

### 회복 방식

- 연애 성공: `+15 ~ +20`
- 결혼/동거 확정: `+25 ~ +35`
- 친한 NPC와 좋은 상호작용: `+5 ~ +10`
- 취미/휴식/외출: `+3 ~ +8`

### 하락 방식

- 하루 기본 감소: `-5`
- 실연: `-15 ~ -20`
- 장기 빈곤/빚 압박: `-5 / day`
- 외출 없음, 고립: `-3 ~ -5 / day`
- 수면 부족: `-5`

## 연애 / 결혼 연결

- 연애는 행복도를 빠르게 끌어올리는 대표 루트다.
- 결혼이나 장기 관계는 엔딩 요약과 랭킹 보정에 직접 반영한다.
- 관계 상태가 깨지면 행복도가 크게 하락하고, 필요하면 `우울한 상태`가 바로 발동할 수 있다.
- 실제 관계 단계와 NPC 반응 규칙은 `docs/design/npc-social-system.md`에서 상세하게 다룬다.

## NPC 연결

- 행복도는 NPC 반응의 유일한 기준이 아니라 보정값으로 쓴다.
- 같은 NPC라도:
  - 행복도가 높으면 대화가 부드럽게 열린다
  - 낮으면 예민한 선택지가 더 자주 열린다
- 친밀한 NPC 이벤트는 행복도 회복 수단으로 쓸 수 있다.

## 엔딩 / 랭킹 연결

### 1차

- 엔딩 요약에 `최종 행복도` 한 줄 추가
- 연애 여부, 결혼 여부, 친한 NPC 수를 행복도 해석 문장에 반영

### 2차

- 랭킹에 `행복` 탭 추가
- 단순 돈 랭킹과 별도로 `행복 점수` 기준 랭킹을 제공

행복 랭킹 계산 예시:

```text
기본 행복 점수
  = 행복도
  + 연애/결혼 보너스
  + 친밀 NPC 보너스
  - 고립/관계 파탄 패널티
```

중요: 현재 온라인 랭킹은 돈 중심이므로, 행복 랭킹은 별도 필드와 정렬 기준을 추가해야 한다.

## 구현 우선순위

1. `state.happiness` 추가
2. 캐릭터 패널에 행복도 바 추가
3. 하루 전환 시 행복도 기본 감소 연결
4. NPC / 연애 이벤트에서 행복도 증감 연결
5. 엔딩 요약에 최종 행복도 반영
6. 이후 행복 랭킹 탭 추가

## 확장 메모

- 나중에 고립, 중독, 실연, 병원, 약물, 정신과 루트가 충분히 쌓이면 `우울증`을 별도 장기 리스크 축으로 분리할 수 있다.
- 그 시점 전까지는 `행복도 + 우울한 상태` 구조를 유지하는 편이 좋다.
