# 전체 시스템 점검 메모

현재 결론: 지금 배금도시는 아이디어 축은 많이 생겼지만, `현실성`, `직업-자산-장소 연결`, `NPC 존재감`을 만드는 핵심 뼈대가 아직 비어 있다. 새 기능을 계속 추가하기 전에, 아래 빈칸을 먼저 메우는 쪽이 전체 완성도가 훨씬 빨리 올라간다.

## 가장 먼저 보이는 큰 문제

### 1. 일자리가 아직 "앱 카드"에 머물러 있다

현재 공고는 폰에서 고르고 미니게임으로 정산하는 감각이 강하다. `실제 회사/매장/현장에 들어가서 일한다`는 생활감이 아직 약하다.

현재 메모: 1차로 `회사 구역`, `오피스플라자`, `관제센터`, `물류허브`, `로비카페`가 지도에 추가됐고 공고 카드에도 `회사명 + 근무지`가 표시된다. 다만 회사 내부 이벤트와 회사 엔티티는 아직 부족하다.

#### 왜 문제인가

- 현실성이 떨어진다.
- 직장과 도시 장소가 예전보다 덜 분리돼 보이지만, 아직 근무지 내부 생활감이 약하다.
- 직업별 삶의 차이가 약하다.

#### 필요한 보강

- 회사/매장/현장 카탈로그
- 출근 장소
- 실제 근무지 배경과 상사/NPC
- 재직중인 회사명 표시
- 회사 내부에서만 뜨는 이벤트

### 2. 자산이 "보유 표시"에 가깝고, 하드 게이트가 약하다

현재 `ownership.vehicle`, `ownership.home`은 있지만, 직업/이동/생활 조건에서 강하게 요구되진 않는다.

#### 지금 이대로 하면 안 되는 이유

- 오토바이/차를 사도 체감이 약해진다.
- 직업 준비의 현실성이 무너진다.
- 배달, 영업, 장거리 이동 루트가 추상적으로 느껴진다.

#### 꼭 넣어야 할 하드 게이트 예시

- 오토바이 또는 최소 자전거가 있어야 특정 배달 가능
- 운전면허가 있어야 차량 기반 직업 가능
- 정장이 있어야 일부 사무/면접 보너스 또는 필수 조건
- 좋은 폰이 있어야 특정 정보직/투자 루트 효율 상승

### 3. 중고거래 / 개인거래 시장이 없다

`호박마켓` 같은 개인간 거래 허브가 없으면 현실성이 크게 빠진다.

#### 빠진 결과

- 안 쓰는 물건을 처분할 수 없다.
- 싼 중고 물건으로 성장하는 루트가 없다.
- 동네 기반 거래, 사기, 득템, 인간관계 이벤트가 비어 있다.

#### 필요 기능

- 중고나라/호박형 앱 또는 동네 게시판
- 직거래 장소
- 흥정 / 사기 / 잠수 / 후기
- 중고 오토바이, 중고폰, 중고가전, 중고옷 거래

### 4. 소비는 커졌는데 유지비 / 감가 / 되팔기가 약하다

집, 차, 옷, 폰을 사는 것만 있고 `유지비`, `수리`, `감가`, `처분`이 없으면 세계가 장난감처럼 느껴진다.

#### 반드시 있어야 할 것

- 자동차: 기름값, 보험, 수리, 주차
- 집: 월세, 관리비, 수리, 압류
- 폰: 통신비, 파손 수리, 배터리 교체
- 옷/명품: 되팔기, 유행, 이미지 변화

### 5. NPC는 상태는 있는데 시스템 레벨의 존재감이 약하다

현재는 `appearance`, `npcs.relations`, `presentation` 기반이 이미 있는데, 플레이 체감상 NPC가 살아 있는 도시를 만들 정도로 연결되진 않았다.

#### 문제

- 어디 가서 누구를 왜 만나야 하는지 모호하다.
- NPC가 안 보이는 순간 도시가 텅 빈다.
- NPC가 돈, 직업, 소비, 사건과 충분히 연결되지 않았다.

#### 먼저 할 것

- 주요 장소마다 최소 1명 보장 스폰
- 역할 태그 표시
- NPC가 주는 정보/위험/루트 명확화
- 랜덤 돌발 이벤트를 NPC가 들고 오게 만들기

### 6. 연애 / 여자친구 시스템이 아직 생활 루프에 안 붙었다

행복도와 attraction 축은 있는데, 플레이어가 실제로 관계를 쌓고 유지하는 루프는 없다.

#### 빠진 감각

- 왜 외모/옷/성형/선물에 돈을 쓰는지 동기가 약함
- 행복도가 숫자 이상으로 느껴지지 않음
- 사람과 사는 세계 느낌이 약함

#### 반드시 필요한 루프

```text
만남
  -> 호감
  -> 연락처
  -> 데이트
  -> 사귀기
  -> 유지 / 이별 / 결혼
```

## 구조적으로 지금 피해야 할 것

### A. 새 직업을 계속 앱 카드로만 추가하는 것

이 방식만 계속 가면 직업은 늘어나는데 세계는 안 커진다.

대신:

- 직업 -> 실제 근무지
- 직업 -> 장비/자격 조건
- 직업 -> 회사/NPC/사건

으로 가야 한다.

### B. 회사 인수를 주식 앱 버튼 하나로 끝내는 것

회사 인수는 단순 매수/매도가 아니라 `기업 엔티티`가 있어야 한다.

필요한 최소 구조:

- 회사 카탈로그
- 지분율
- 경영권 상태
- 인수전 이벤트
- 매각 후 결과

즉 `주식앱 안의 특수 버튼`으로 끝내면 안 되고, `business` 도메인으로 빼야 한다.

### C. 배달 현실성을 "수익 보너스 아이템"만으로 처리하는 것

현재처럼 자전거/장비가 수익 보너스만 주는 식으로 가면 현실성이 약하다.

더 맞는 방식:

- 오토바이/자전거가 없으면 해당 배달 자체가 잠김
- 등급 좋은 탈것이면 효율 보너스
- 날씨/정비 상태가 결과에 반영

### D. NPC를 대화 텍스트 덩어리로만 늘리는 것

NPC 수만 늘리고 역할을 안 주면 더 밋밋해진다.

NPC마다 반드시 있어야 할 것:

- 기능
- 감정 변화
- 사건
- 관계 보상

### E. 같은 개념을 서로 다른 모듈이 따로 들고 가는 것

지금은 `배달`이 한쪽에선 소비 앱이고, 다른 한쪽에선 직업이다. 또 `shop.js`는 별도 독립 모듈인데 메인 구조와 아직 연결되지 않았다.

이 상태로 계속 기능을 붙이면:

- 이름은 같은데 전혀 다른 시스템이 생긴다.
- 어떤 파일이 기준인지 헷갈린다.
- 나중에 시설/상점/중고거래를 붙일 때 충돌이 커진다.

즉 앞으로는:

- 소비 앱
- 직업
- 상점 / 거래
- 시설

를 도메인별로 분리하고, 미연결 임시 모듈은 빨리 흡수하거나 폐기하는 편이 맞다.

## 추가하면 좋은 현실성 요소

### 1. 회사/매장 실체화

- 편의점 본사
- 카페 체인점
- 물류센터
- 학원
- 병원
- 백화점
- 증권사

플레이어는 "어디 업장 사람인지"가 보여야 한다.

### 2. 중고거래 허브

- 호박마켓 앱
- 동네 직거래
- 사기 / 노쇼 / 흥정
- 중고 오토바이, 폰, 정장, 가전

### 3. 자격증 / 허가증 계층화

- 운전면허
- 2종 소형 또는 오토바이 면허
- 컴퓨터 자격
- 공인중개 보조 자격
- 투자/법인 관련 상위 루트 자격

### 4. 직업별 필수 준비물

| 직업 | 필수 조건 예시 |
| --- | --- |
| 배달 | 자전거/오토바이 + 면허 |
| 사무직 | 정장 + 컴퓨터 자격 |
| 영업직 | 차량 또는 외형/평판 기준 |
| 과외 | 지능 + 태블릿/노트 |
| 현장직 | 체력 + 작업복 |

### 5. 거래/재판매 루프

- 산 물건을 다시 판다
- 감가가 붙는다
- 급전 필요할 때 자산을 처분한다
- 중고 거래로 사기/인맥/NPC 이벤트가 발생한다

## 추천 구현 우선순위

1. `실제 회사/근무지 모델`
2. `하드 게이트 장비/면허 조건`
3. `호박마켓 같은 중고거래 시장`
4. `자산 유지비 / 수리 / 되팔기`
5. `NPC 스폰 안정화 + 역할 가시화`
6. `연애 / 여자친구 루프`
7. `회사 인수 / 매각`

## 한 줄 요약

지금은 기능을 더 늘리는 것보다, `장소`, `회사`, `장비`, `NPC`, `관계`가 실제로 서로 물리게 만드는 쪽이 훨씬 중요하다.
