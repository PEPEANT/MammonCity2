# 출생 자산 / 현금 / 은행 흐름

현재 결론: 수저 시작 차이는 `손에 들고 있는 현금`보다 `출생 자산 패키지`, `계좌 잔액`, `가족 안전망`, `초기 소유품`으로 나누는 편이 현실적이다. 게임 내에서 바로 쓰는 돈은 `현금`, 급여와 큰돈은 `은행 잔고`로 분리한다.

## 목적

- 수저별 출발선 차이를 `현금 숫자` 하나가 아니라 실제 생활 조건 차이로 보이게 만든다.
- 월급, 대출, 담보, 자산 매매, 생활 소비가 같은 돈 흐름 위에서 움직이게 만든다.
- 차량과 집을 단순 소유품이 아니라 `이동`, `직업`, `대출`, `매각`과 연결한다.
- 길거리 소비와 계좌 기반 거래를 분리해 은행 앱의 의미를 키운다.

## 핵심 결정

- `state.money`는 손에 들고 있는 `현금`이다.
- `state.bank.balance`는 계좌에 있는 `은행 잔고`다.
- 월급, 급여성 보상, 대출금, 자산 매각금은 기본적으로 `은행 잔고`로 들어간다.
- 편의점, 길거리 음식, 소액 도박, 현장 잡비는 `현금`으로 우선 지불한다.
- 차량, 집, 큰 소비, 대출 상환은 기본적으로 `은행 잔고`를 우선 사용한다.
- 수저는 `초기 현금`보다 `출생 자산 패키지`가 더 중요하다.

## 돈 흐름 한눈에 보기

```text
출생 패키지
  -> 현금
  -> 계좌 잔고
  -> 초기 소유 자산
  -> 가족 안전망

알바/월급/거래/대출
  -> 은행 잔고로 입금

은행 앱
  -> 현금 인출
  -> 현금 입금
  -> 대출 실행
  -> 상환

생활 소비
  -> 현금 우선

자산 소비 / 계약 / 담보
  -> 은행 잔고 우선
```

## 출생 자산 패키지 원칙

수저별 차이는 아래 네 축으로 준다.

1. 손에 쥔 현금
2. 계좌에 있는 시작 잔고
3. 바로 쓰거나 팔 수 있는 소유 자산
4. 급할 때 버티게 해주는 가족 안전망

### 권장 기준

| 등급 | 현금 | 계좌 잔고 | 시작 자산 예시 | 안전망 |
| --- | ---: | ---: | --- | --- |
| 금수저 | 300,000원 | 12,000,000원 | 차량 접근권 또는 중형차, 고급 스마트폰, 좋은 거처 | 1회 큰 지원 |
| 은수저 | 150,000원 | 4,000,000원 | 중고차 또는 오토바이, 괜찮은 스마트폰, 안정 거처 | 1회 중간 지원 |
| 동수저 | 70,000원 | 800,000원 | 자전거 또는 중고 전자기기, 기본 거처 | 소액 지원 |
| 쇠수저 | 40,000원 | 150,000원 | 자산 없음 또는 소형 생활가전 1개 | 거의 없음 |
| 흙수저 | 20,000원 | 0원 | 자산 없음, 경우에 따라 소액 채무 압박 | 없음 |

위 수치는 시작점 초안이다. 중요한 건 `금수저는 통장과 자산이 두껍고`, `흙수저는 현금도 잔고도 없으며 팔 수 있는 것도 거의 없다`는 체감이다.

## 출생 자산 패키지 메모

- 최상위 수저에게 `펜트하우스 직접 소유`를 바로 주는 건 과하다.
- 대신 `좋은 거처`, `가족 명의 차량 사용`, `큰 계좌 잔고`, `프리미엄 소비 접근` 쪽이 현실적이다.
- 흙수저는 시작 현금만 적은 게 아니라 `계좌 잔고`, `자산`, `안전망`이 모두 비어 있어야 체감된다.
- 수저 결과는 앞으로 `호박마켓 고급 매물 노출`, `대출 한도`, `가족 지원 이벤트`에도 연결할 수 있다.

## 현금과 은행 분리 규칙

### 현금으로 보는 것

- UI 상단에 바로 보이는 보유 돈
- 편의점, 길거리 음식, 병원 잡비, 교통 잡비
- 소규모 사행성 소비
- 일부 호박마켓 직거래

### 은행 잔고로 보는 것

- 월급, 일급 정산, 보너스
- 대출 실행 금액
- 집 계약금, 차량 구매 대금
- 월 상환금, 자동이체, 관리비/월세
- 자산 매각 대금

### 입금/출금 규칙

- 월급은 무조건 계좌 입금
- 플레이어가 은행 앱에서 `현금 인출`해야 생활비로 쓸 수 있음
- 현금이 남으면 다시 `입금` 가능
- 계좌에 돈이 있는데 현금이 없으면 길거리 소비를 못하는 상황이 일부 발생하게 둔다

## 자산 구매 / 매각 규칙

### 차량

- 중고 구매는 `호박마켓` 또는 자동차매장
- 구매 대금은 기본적으로 계좌에서 빠짐
- 일부 직거래는 현금 거래 선택 가능
- 매각 대금은 기본적으로 계좌에 입금
- 차량은 이동 시간, 배달 직업, 담보 대출과 연결
- 현재 보유 차량은 `ownership.vehicleAsset`에 매입가와 현재 평가액까지 같이 남긴다

### 집

- 계약/구매는 계좌 거래가 기본
- 월세/보증금/매매대금은 현금보다 계좌 기준이 자연스럽다
- 집을 잃으면 `state.ownership.residence`를 하위 거처로 강등
- 집은 담보 대출과 직접 연결
- 현재 보유 집은 `ownership.homeAsset`에 매입가와 현재 평가액까지 같이 남긴다

## 월급과 생활비 흐름

권장 루프:

1. 출근/근무 완료
2. 월급 또는 일급이 `state.bank.balance`로 입금
3. 플레이어가 필요할 때 현금을 인출
4. 길거리 소비는 현금으로 지출
5. 턴 시작에 대출 상환과 정기비용이 계좌에서 자동 차감

이렇게 가면 `통장에 돈은 많은데 현금이 없어서 당장 밥을 못 사는` 상황도 만들 수 있고, 반대로 `손에 쥔 현금은 많지만 계좌가 말라서 자동이체가 터지는` 상황도 만든다.

## 상태 구조 초안

```js
state.money = 70000;

state.bank = {
  balance: 800000,
  transactions: [],
  transferDraft: {
    recipient: "",
    amount: "",
  },
  loans: [],
};

state.startTier = {
  id: "bronze",
  label: "동수저",
  walletCash: 70000,
  bankBalance: 800000,
  starterAssetIds: ["bicycle"],
  safetyNetLevel: "small",
  applied: false,
};
```

`state.ownership`는 여전히 집/차 원본을 갖고, `state.inventory`는 문서/소지품 표시를 맡는다.

현재 구현 기준:

- 차량/집 매입은 계좌 기준으로 처리된다.
- 차량/집 매각 대금도 계좌로 들어간다.
- 소액 생활 자산은 여전히 현금 직거래가 가능하다.
- 호박마켓과 은행 담보 평가는 같은 `현재 평가액`을 읽는다.

## 구현 우선순위

1. 수저 시작값을 `현금 + 계좌 잔고 + starterAssetIds` 구조로 바꾼다.
2. 월급/일급 지급처를 `현금`에서 `은행 잔고`로 옮긴다.
3. 은행 앱 입출금이 실제 생활 루프에 영향을 주게 만든다.
4. 호박마켓 차량/집 거래를 계좌 결제 기준으로 바꾼다.
5. 대출/담보/압류를 그 위에 얹는다.

## 리스크

- 은행 잔고와 현금을 무조건 둘 다 요구하면 초반이 지나치게 답답해질 수 있다.
- 수저별 자산 격차가 너무 크면 출생 결과가 플레이보다 중요해질 수 있다.
- 차량/집을 바로 소유시키면 밸런스가 튈 수 있으니 `가족 차량 접근`, `좋은 거처`, `프리미엄 매물 접근` 같은 완화형도 고려해야 한다.
