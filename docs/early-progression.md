# 초반 진행 구조

현재 확정 규칙: 게임은 30일 동안 진행되며, 7일차에 동네 알바 시장이 무너지는 이벤트가 시작되고, 이후에는 선택 분기 없이 도시맵 중심 흐름으로 넘어간다. 알바는 직접 버튼으로 고르는 방식이 아니라 휴대폰으로 공고를 보고 지원하는 방식으로 진행한다.

## 목적

이 문서는 구현 전에 초반 구조를 먼저 고정하기 위한 문서다.
핵심은 기존의 `오늘 알바를 바로 고른다`는 느낌을 없애고, 아래 흐름으로 바꾸는 것이다.

`휴대폰 -> 알바 공고 확인 -> 신청 -> 면접 -> 합격/불합격 -> 다음날 출근 -> 근무 진행`

## 핵심 흐름

### 현재 확정 규칙

1. 전체 플레이 기간은 30일이다.
2. 초반에는 시작 분기가 없다.
3. 초반 진행은 하나의 도시 진입 흐름으로 이어져야 한다.
4. 스마트폰은 1일차 프롤로그가 끝난 뒤 바로 사용할 수 있다.
5. 7일차가 첫 번째 큰 이야기 이벤트다.
6. 7일차 이후 동네 알바 공고는 급격히 줄어든다.
7. 이 공고 감소가 주인공이 도시로 나가기로 결심하는 계기가 된다.
8. 메인 UI에서 오늘의 알바를 직접 고르게 하면 안 된다.
9. 알바는 반드시 휴대폰 기반 지원 절차를 통해 얻어야 한다.

### 하루 진행 구조

1. 아침이 시작된다.
2. 플레이어가 휴대폰을 연다.
3. 알바 공고를 확인한다.
4. 원하는 공고에 지원한다.
5. 면접이 잡히거나 바로 면접 결과가 처리된다.
6. 합격하면 다음날 출근 일정이 생성된다.
7. 다음날이 되면 해당 근무지에 실제로 가야 한다.

### 초반 타임라인

1. 1일차부터 6일차까지
   플레이어는 동네에서 기본 생존 루프를 익힌다.
   1일차 프롤로그 종료 후 바로 스마트폰 공고 앱을 사용할 수 있다.
   이 시기에도 알바는 직접 선택이 아니라 휴대폰 공고를 통해 연결되어야 한다.

2. 7일차
   주요 스토리 이벤트가 시작된다.
   동네 알바 시장이 줄어드는 변화가 플레이어에게 분명하게 체감되어야 한다.

3. 7일차 이후
   동네 알바 공고 수가 크게 줄거나 조건이 나빠진다.
   플레이어가 현재 환경으로는 더 이상 안정적으로 버티기 어렵다는 압박을 느껴야 한다.

4. 도시 진입 이후
   주인공은 도시로 나가겠다고 결심한다.
   이후 메인 플레이 공간은 도시맵 하나로 고정한다.

## 제약 조건

- 초반 루프를 `오늘 알바 하나 고르기`처럼 보이게 만들면 안 된다.
- 알바 탐색의 시작점은 휴대폰이어야 한다.
- 면접 결과는 의미가 있어야 하며, 그래야 일자리를 얻는 감각이 생긴다.
- 합격한 알바는 즉시 시작하지 않고 반드시 다음날 출근으로 이어져야 한다.
- 7일차 이벤트는 플레이어 상황을 실제로 바꾸는 사건이어야 한다.

## 아직 미확정인 내용

- 도시로 나갈 때 엄마가 지원금을 주는지 여부
- 7일차 이후 정확히 며칠째 도시로 넘어가는지
- 하루에 알바 지원을 몇 번까지 허용할지
- 면접을 즉시 처리할지, 별도 장면으로 보여줄지

## 현재 구현 메모

- 현재 1차 구현은 `공고 앱 -> 지원 -> 즉시 면접 결과 -> 다음날 예약 출근`까지 연결된 상태다.
- 예약된 근무는 다음날 방 화면과 공고 앱에서 시작할 수 있다.
- 기존 사건/결과 씬은 예약 출근 이후의 실제 근무 장면으로 재사용한다.
