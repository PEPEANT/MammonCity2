# 초반 진행 구조
현재 결론: 지금 플레이 가능한 범위는 `12턴 프로토타입`이며, 1턴은 1개월로 취급한다. 시작 오프닝 이후 `단기알바 생존 루프`와 `직장 면접 준비 루프`가 함께 열린다.

## 목적

현재 코드 기준으로 초반 하루 흐름이 어디까지 구현됐는지 빠르게 파악하기 위한 문서다.

## 현재 코어 루프

1. 1일차 프롤로그와 방청소를 진행한다.
2. 방에서 스마트폰과 기본 UI를 확인한다.
3. 밖으로 나가 버스정류장과 도시 장소를 이동한다.
4. 공고 앱에서 `단기알바`로 생활비를 버티거나, `직장` 면접을 위한 준비를 한다.
5. 남은 시간은 이동, 대기, 기억 로그 확인, 인벤토리 확인에 쓴다.
6. 12턴이 끝나면 최종 정산과 랭킹으로 넘어간다.

## 현재 장소 기반 준비 루프

### 버스정류장 지도

현재 day01 버스 이동 화면에서 갈 수 있는 핵심 장소:

- `집앞 골목`
- `버스 정류장`
- `도시 사거리`
- `역 앞`
- `번화가`
- `맥도날드`
- `도서관`
- `시험장`
- `배금디지털단지`
- `배금디지털단지 입구`
- `배금전자 사무동`
- `배금전자 생산동`
- `배금연구소`

### 도서관

- 사무 준비도 상승
- 학업 준비도 상승

### 맥도날드

- 초기 문서에 남아 있던 `오픈 카페` 자리는 현재 `맥도날드 배금거리점`으로 정리한다.
- 현재 venue는 `mcdonalds -> mcdonalds-counter -> mcdonalds-kitchen` 3단 구조다.
- 손님 행동은 카운터에 남기고, 알바는 `카운터 / 조리` 2포지션으로 분리돼 있다.

### 시험장

- 컴퓨터 자격 획득
- 운전면허 획득

## 직업 루프

### 단기알바

- 하루 1회 지원 가능
- 현재 라이브 규칙은 사실상 `에너지 > 0 = 바로 예약근무 생성`에 가깝다.
- `에너지 = 0`일 때는 장기적으로 `불합격`보다 `지원 불가` 문구로 정리할 예정이다.
- 다음 턴 예약 출근은 근무지에 실제로 도착해야 시작할 수 있다.
- 맥도날드는 `카운터 / 조리` 2포지션이 분리된 첫 근무지 사례다.

### 직장

- 준비도와 자격 조건 확인
- 하루 1회 면접 신청
- 다음 날 면접 결과 확인
- 합격 시 장기 루트와 추가 공고 해금
- `생산직 / 사무직 / 연구직` 3개 공고가 열린다.
- 세 공고 모두 `대학 졸업` 이후 면접 신청 가능
- 사무직과 연구직은 `컴퓨터 자격`, `정장`, `지능` 같은 하드 게이트를 요구
- 예약된 근무는 근무지에 도착해야 실제 출근 가능

## 현재 상태 메모

- `MAX_DAYS = 12`
- `jobs.activeTrack`으로 공고 앱 탭 유지
- `jobs.careerPrep`으로 준비도 저장
- `jobs.certifications`로 자격 저장
- `jobs.career`로 직장 진행 상태 저장
- 캐릭터 패널은 `생활 / 사회 / 직업 / 생활수준` 통합 스탯 묶음으로 표시

## 아직 없는 것

- 대학가 심화 이벤트/NPC
- 공원 기반 회복/NPC 허브
- 면접 장면과 재직 유지 시스템
- 범죄자/창업가/투자자 이벤트 루트
