# 스마트폰 시스템

현재 확정 규칙: 스마트폰은 해금 이벤트 없이 1일차 시작부터 바로 사용할 수 있으며, 우측 폰 화면 안에서 `공고` 앱으로 지원, 면접 결과 확인, 다음날 출근 예약까지 처리한다.

## 목적

이 문서는 게임 내 스마트폰의 해금 시점과 역할을 먼저 고정하기 위한 문서다.
스마트폰은 초반 알바 탐색의 핵심 도구이며, 이후 알바 시스템 개편의 시작점이 된다.

## 현재 확정 규칙

1. 스마트폰은 1일차 시작부터 바로 사용할 수 있다.
2. `3일차 아침 -> 방청소 -> 부모님이 압수당했던 스마트폰을 돌려준다` 흐름은 삭제 상태로 유지한다.
3. 스마트폰 해금을 별도 이벤트로 늦추지 않는다.
4. 초반 알바 탐색은 스마트폰을 기준으로 설계한다.
5. 우측 폰 패널은 실제 작은 스마트폰 화면처럼 보이게 유지한다.
6. 앱 사용 결과는 폰 화면 안 프리뷰와 헤드라인 둘 다에 반영한다.
7. 스마트폰은 기본 숨김 상태로 두고, `폰 열기` UI 버튼으로 다시 띄운다.
8. `공고` 앱은 폰 안의 별도 화면으로 열린다.
9. 공고 지원 결과는 같은 날 바로 도착하고, 합격 시 다음날 출근이 예약된다.
10. 예약된 근무는 다음날 방 화면이나 공고 앱에서 바로 출근할 수 있다.

## 해금 흐름

### 기존안

```text
3일차 아침
  -> "세 번째 아침, 방이 또 어질러져 있다"
  -> 방청소를 한다
  -> "압수당했던 스마트폰을 돌려받았다"
  -> "스마트폰을 켠다"
  -> 스마트폰 사용 가능
```

### 수정안

```text
1일차 시작
  -> 스마트폰이 이미 손에 있음
  -> 우측 폰 화면에서 앱 확인
  -> 공고 앱에서 오늘 공고 확인
  -> 지원
  -> 면접 결과 확인
  -> 합격 시 다음날 출근 예약
```

즉, 스마트폰은 1일차 아침부터 이미 손에 있는 상태로 본다.

## 시스템 역할

스마트폰은 단순한 장식이 아니라 아래 기능의 진입점이 된다.

1. 알바 공고 확인
2. 알바 신청
3. 생활 앱 사용 결과 확인
4. 면접 결과 확인
5. 다음날 출근 일정 확인

## 연동 기준

초기 구현 목표는 아래 흐름이다.

```text
스마트폰 화면 확인
  -> 알바 공고 앱 진입
  -> 오늘 올라온 공고 확인
  -> 지원
  -> 면접 결과 즉시 처리
  -> 합격 시 다음날 출근 일정 생성
  -> 다음날 공고 앱 또는 방 화면에서 출근 시작
```

## 제약 조건

- 스마트폰 사용 시작 시점을 1일차 이후로 미루지 않는다.
- 부모님이 돌려주는 연출은 현재 기준에서 제거한다.
- 방청소 선택과 스마트폰 해금을 연결하지 않는다.
- 스마트폰은 알바 시스템과 바로 이어질 수 있어야 한다.
- 폰 패널 안에서 앱 버튼과 결과 프리뷰가 동시에 보여야 한다.
- 숨김 상태에서는 본문과 씬 영역이 다시 넓어져야 한다.
- `공고` 앱은 열람만으로 닫히지 않고, 같은 날의 지원 결과와 예약 상태를 계속 보여줄 수 있어야 한다.
