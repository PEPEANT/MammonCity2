# Stats Balance Feedback Plan
현재 결론: 다음 밸런스 패스는 `배고픔 100 스케일`, `스탯 변경 공용 경로`, `중앙 하단 액션 피드`를 한 번에 묶어서 정리하는 것이 맞다.

## Purpose

- 첫 베타 기준에서 스탯과 생존 규칙을 단순하게 만든다.
- 행동 결과가 한눈에 읽히도록 UI 피드백을 공용 구조로 통일한다.
- 기존 세이브가 있어도 수치가 안전하게 보정되도록 마이그레이션한다.

## Current Rules

- 체력과 에너지는 0~100 보정이 이미 들어가 있다.
- 배고픔은 아직 `HUNGER_MAX = 5`, `HUNGER_DECAY_INTERVAL_MINUTES = 4.5 * 60` 기준이다.
- 공부, 식사, 자격증, 아이템 사용 결과는 대부분 `headline + memory` 조합으로만 표시된다.
- 공부/준비 행동은 `logic.js` 내부에서 직접 수치를 바꾸고, 아이템/식사는 다른 함수에서 따로 처리한다.

## Current Problems

1. 배고픔이 5칸이라 플레이어가 체감하기 어렵고, 회복/감소 폭이 너무 거칠다.
2. 스탯 변경 경로가 여러 파일에 흩어져 있어 밸런스 조정 시 누락되기 쉽다.
3. 행동 결과가 중앙 피드 없이 `headline`에만 남아서 무엇이 올랐는지 바로 읽히지 않는다.
4. `공부했다`, `먹었다`, `샀다`, `자격증을 땄다` 같은 행동이 모두 같은 톤으로 보여 게임 감각이 약하다.

## 단기알바와 Hunger V3 연결 기준

- 단기알바는 `합격/불합격 RNG`보다 `에너지/배고픔 상태`로 읽히게 가야 한다.
- 즉 Hunger V3 이후에는 `배고픔 저하 -> 에너지 회복 악화 -> 알바 지원/근무 부담 증가` 흐름이 보여야 한다.
- 단기알바 지원이 막히는 경우도 `채용 실패`보다 `에너지 부족`, `컨디션 부족`처럼 needs UI로 설명하는 편이 맞다.
- 맥도날드 카운터/주방은 이 기준을 가장 먼저 검증할 장소다.

## Work Order

### 1. Stat Mutation Audit

- `체력 / 에너지 / 배고픔 / 행복도 / 지능 / 평판 / 범죄도 / 외모도`의 직접 변경 지점을 수집한다.
- 직접 대입을 줄이고 공용 helper를 만든다.
- 목표 helper
  - `applyPlayerNeedDelta(...)`
  - `applyPlayerSocialDelta(...)`
  - `queueGameplayFeedback(...)`
- 이 단계에서 상한/하한 보정은 helper 안으로 몰아넣는다.

### 2. Hunger V3

- `HUNGER_SYSTEM_VERSION`을 올리고 구세이브를 비율 기반으로 변환한다.
- `HUNGER_MAX`를 100으로 바꾼다.
- 시작 배고픔은 항상 100으로 둔다.
- 감소는 `한 번 굶었다고 바로 죽지 않는 속도`로 다시 맞춘다.
- 첫 구현 목표
  - 하루를 통째로 보내도 즉사하지 않는다.
  - 식사 1~2번이 의미 있게 느껴진다.
  - 편의점/패스트푸드/인벤토리 음식 회복량을 100 스케일로 다시 맞춘다.
- 경고 구간도 새로 둔다.
  - 60 이하: 공복
  - 30 이하: 위험
  - 10 이하: 응급 경고

### 3. Bottom-Center Feedback UI

- 화면 중앙 아래에 짧게 뜨는 공용 액션 피드를 만든다.
- 한 액션은 아래 정보를 가질 수 있다.
  - 제목: `도서관에서 공부했다`
  - 변화칩: `지능 +2`, `사무 준비도 +1`, `에너지 -8`
  - 획득/소비: `삼각김밥 획득`, `1,800원 사용`
- `headline`은 장면 설명용으로 두고, 수치 변화는 액션 피드가 담당한다.
- 피드는 queue 구조로 두고 짧은 시간 뒤 자동 사라지게 한다.

### 4. First Adoption Targets

- 도서관 공부
- 취업지원센터 상담
- 시험장 자격증 취득
- 편의점 구매
- 맥도날드 식사
- 인벤토리 음식 사용
- 단기알바 지원/출근 시작 피드

이 일곱 군데가 먼저 정리되면 베타 기준 체감이 가장 커진다.

### 5. Verification

- 새 게임 시작 시 배고픔이 100으로 시작하는지 확인
- 기존 세이브 로드 시 배고픔이 비정상 값으로 깨지지 않는지 확인
- 공부 1회 후 `지능/준비도 증가`, `에너지 감소`가 피드에 같이 뜨는지 확인
- 음식 구매 후 `아이템 획득` 피드가 뜨고 실제 인벤토리에 들어가는지 확인
- 음식 사용 후 `배고픔 회복` 수치가 100 상한을 넘지 않는지 확인
- 응급/병원 분기가 여전히 동작하는지 확인

- 전 직업 미니게임 결과 카드 통합
